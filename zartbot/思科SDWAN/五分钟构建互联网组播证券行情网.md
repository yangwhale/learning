### 组播行情网络 

组播技术**其实一直比较小众，最大的原因就是Internet上开展组播技术非常困难。因此这个技术最常见的场景就是`IPTV`和`证券行情`.当然今天还是来讲讲如何利用SDWAN实现互联网组播。有需要的券商和交易所可以小窗后台联系我~

其实我国资本市场建立之初，行情和交易传输就是采用`云`的方式，即利用卫星通信的方式，即各地建立卫星小站接收行情。然后伴随着各种广域网专线技术的出现，逐渐改造为采用组播传递的地面高速行情专网。

通常是各个会员机构通过专线连接到交易所，由交易所边界路由器组播复制给各个会员。券商通常需要在自建的广域网汇聚节点上接收来自交易所的行情，并将复制其分发给各个营业厅。

这种多级中心化的复制方式对汇聚点带宽需求非常高，同时也容易产生单点故障，组播行情灾备设计较为困难。另一方面，由于行情都需要绕行到广域网汇聚节点再转发，部分地域分支机构行情延迟较大。

如果交易所或者券商构建地域性的节点又会面临远端站点管理运维费用较高的问题...

那么... 能不能借用公有云来实现呢？国内大多数运营商连接到各个Region的公有云带宽和服务质量都是有保证的，那么我们是否可以在公有云上构建一些区域性的组播复制节点，然后利用这些节点来就近将行情源转发给一些机构呢？

### SDWAN基于地域的组播复制 

今天我们就来做这样一个实验.使用SDWAN的方式可以构建基于混合云的组播行情网络，通过使用部署在公有云上的虚拟路由器**，构建区域性的行情转发中心，借助公有云的带宽和高质量链路资源进行行情组播转发，本次测试我们在阿里云北京、上海、杭州、深圳、河源、张家口、河源、成都、建立了8台虚拟路由器构建混合云行情专网。

在每个区域性的节点配置如下：

```
ip pim vrf 1 autorp listenerip pim vrf 1 spt-threshold 0ip pim vrf 1 send-rp-discovery Loopback1 scope 255ip pim vrf 1 ssm defaultip multicast-routing vrf 1 distributed!sdwan multicast  address-family ipv4 vrf 1   replicator
```

而在会员接入的路由器只需要配置AutoRP即可:

```
ip pim vrf 1 autorp listenerip pim vrf 1 spt-threshold 0ip multicast-routing vrf 1 distributed!interface GigabitEthernet5no shutdownvrf forwarding 1ip address 10.20.30.40 255.255.255.0ip pim sparse-modeip igmp version 3!sdwanmulticast  address-family ipv4 vrf 1
```

然后骚操作来了，路由器居然会根据区域性节点GPS和自己的GPS经纬度信息自己找最近的RP...
例如我们模拟一台重庆的路由器，它会自动找到阿里云成都节点

### 五分钟构建全国大网 

您可以去如下连接下载初始化工具

```
https://github.com/zartbot/sdwan_aliyun_init
```

在部署前，请准备好您安装Vmanage所使用的root-ca-chain.pem以及按照如下格式配置您的vmanage信息，并将其保存为`vmanage.yml`

```
baseurl: <vmanageIP:8443>username: adminpassword: AdminPassword
```

然后按照如下格式填写安装的CSR**区域和相应的配置

```
regionId,imageID,zoneId,lat,long,securitygroup,wanSwitch,WANIP,lanSwitch,LANIp,Lo,serviceVPN,siteid,siteip,hostname,loginpassword,bwcn-shanghai,m-uf6hwh6xxxxx,cn-shanghai-l,31.232,121.469,sg-123456,vsw-123456,10.116.0.1,vsw-123456,10.116.1.1,10.99.1.116,99,116001,10.116.0.1,ali_shanghai1,adminpwdhaha,1cn-hangzhou,m-xxxxxx,cn-hangzhou-j,1.2,2.3,sg-123456,vsw-123456,10.117.0.1,vsw-123456,10.117.1.1,10.99.1.117,99,117001,10.117.0.1,ali_hangzhou1,adminpwdhaha,1
```

然后执行`sdwan_aliyun_init`

```
./sdwan-aliyun_linux -cert=root-ca-chain.pem -config=aliyun.csv -vmanage=vmanage.yml
```

它会输出aliyun cloud-shell的命令行， 如果您本地安装了aliyun可执行文件和配置好了相应权限的token直接执行即可， 如果没有，可以在阿里云控制台打开CloudShell直接黏贴进去。然后等五分钟，覆盖全国的组播行情网络就上线了。
