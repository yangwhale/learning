**题记：其实很多客户都有一些需求，希望将一部分没有弹性的业务部署在本地，然后借助云的出口提供业务访问。我们接下来这一个文章以思科SDWAN控器在本地部署，但通过阿里云公网出口提供服务为例，给大家介绍这个场景。**

最近很多客户在问我要把SDWAN的控制器放置在阿里云上，当然技术上是没问题的，至少现在您就可以拿qcow2的镜像去部署，但是这是**非官方支持的**。官方支持需要考虑在云的环境中控制器的安全验证，以及大量的读写I/O的保障，这些还需要一定的时间去协商，因此思科现阶段在其它公有云上都是采用思科自己DevOps的方式帮助客户运维。

但是我仔细了解了客户需求后，本质上只是很多客户本地没有静态公网IP资源，需要使用公有云资源。这样很多中小客户被迫为了静态IP租用IDC机房**服务器，或者拉专线，有没有更便宜的做法呢？我计算了一下三个控制器(vManage/vSmart/vBond)在公有云上的成本，而且这些节点本身也是没有弹性的需要一直开着，很多运维数据需要弄到线下来，那么是否可以采用另一种方案呢？我们通过VPN连接到云端，然后仅使用公有云的静态IP地址**资源和出口资源，控制器还是安装在本地？这就是今天要讲的方案。

在VPN使用上，我们做了一些特殊的处理，Gi1我们都放置在全局路由表中，而Gi2我们都将其放置在一个特殊的Local VRF中，这样可以做到内网外网的完全隔离。

阿里云隧道配置：

```
vrf definition ServiceLAN  address-family ipv4!  interface Loopback0 vrf forwarding ServiceLAN  ip nat inside ip address 172.16.0.1 255.255.255.255!interface GigabitEthernet2 vrf forwarding ServiceLAN ip address dhcp ip nat outside!crypto ikev2 authorization policy cloudL2_Auth pool FlexVPNSpokes route set interface!crypto ikev2 profile cloudL2 match identity remote fqdn domain cisco.com identity local fqdn AliCloudL2.cisco.com authentication remote rsa-sig authentication local rsa-sig pki trustpoint CA aaa authorization group cert list default cloudL2_Auth virtual-template 1!crypto ipsec profile cloudL2 set ikev2-profile cloudL2!interface Virtual-Template1 type tunnel vrf forwarding ServiceLAN ip unnumbered Loopback0 ip nhrp network-id 1234 ip nhrp redirect ip tcp adjust-mss 1360 ip nat inside tunnel source GigabitEthernet1 tunnel mode ipsec ipv4 tunnel path-mtu-discovery tunnel protection ipsec profile cloudL2!ip local pool FlexVPNSpokes 172.16.0.100 172.16.0.254
```

**2. 配置路由和NAT**

云端连接业务的Gi2弹性网卡添加3个辅助IP用于SDWAN控制器，配置如下：

![图片](assets/52f18f1e69d5.png)

由于IPSecVPN用的主网卡和辅助网卡通过不同的VRF**隔离开了，因此在同一个网段也没关系。路由协议配置如下

```
router eigrp 100 ! address-family ipv4 vrf ServiceLAN  redistribute connected  redistribute static  network 172.16.0.0 0.0.0.255  autonomous-system 100 exit-address-family!ip nat inside source static 10.88.1.2 10.88.0.2 vrf ServiceLANip nat inside source static 10.88.1.3 10.88.0.3 vrf ServiceLANip nat inside source static 10.88.1.4 10.88.0.4 vrf ServiceLAN
```

本地数据中心

```
router eigrp 100 ! address-family ipv4 vrf ServiceLAN  redistribute connected  network 172.16.0.0 0.0.0.255  autonomous-system 100 exit-address-family
```
