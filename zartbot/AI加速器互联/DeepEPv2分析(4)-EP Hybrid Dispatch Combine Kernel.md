## 1. Hybrid Dispatch

Hybrid Dispatch和Direct Dispatch的区别在于Hybrid需要同时使用ScaleUP NVLink和ScaleOut RDMA, 主要用于多卡多机的场景.

项目

Direct Dispatch

Hybrid Dispatch

适用拓扑
`num_scaleout_ranks == 1``num_scaleout_ranks > 1`
(跨机)

Rank 空间

单维 `rank_idx`

二维 `scaleout × scaleup`

Warp 角色数

2 种(Notify / Dispatch)
**3 种**
(Notify / Scaleout-Send / Forward)

Buffer 段数

2 段(recv / send)
**3 段**
(scaleup / scaleout_send / scaleout_recv)

数据流跳数

1 跳(直达 peer recv)
**2 跳**
(RDMA → scaleout_recv → NVLink → scaleup)

通信语义

LSA 或 World

Rail(RDMA 跨机)+ LSA(NVLink 机内)

触发 epilogue
`cudaTriggerProgrammaticLaunchCompletion`

注意
Hybrid Dispatch数据流类似于DeepEPv1中的Normal Kernel , 对于大EP构建多机 Decoding 的场景(在DeepEPv1中的LowLatency Kernel)可以选择`allow_hybrid_mode == false`时, num_scaleout_ranks=1, 但是num_scaleup_ranks = num_rdma_ranks * num_nvl_ranks, 此时依旧会使用 direct 模式.

        // ========== 角色 2: Scale-out send warp ==========        const int scaleout_warp_idx = warp_idx - kNumNotifyWarps;        // channel = sm * kNumChannelsPerSM + local_warp；每个 warp 独占一个 channel        const int channel_idx       = sm_idx * kNumChannelsPerSM + scaleout_warp_idx;        // 从 scaleout_recv_buffer 切出 (rank=self, channel=channel_idx) 的视图        //   本 scaleout 内 bypass 的 token 会直接 TMA store 到这里        scaleout_recv_buffer = scaleout_recv_buffer.get_rank_buffer(scaleout_rank_idx);        scaleout_recv_buffer = scaleout_recv_buffer.get_channel_buffer<kNumMaxTokensPerChannel>(channel_idx);        // Channel metadata maintenance        // ★ 累计式 tail 通告: 每发 kScaleoutUpdateInterval 个 slot 才把 tail red_add 给对方        //   减少通信频次；finish_flag 打包在高位作为 "本 channel 已完成"        EP_STATIC_ASSERT(kNumScaleoutRanks <= 32, "Invalid number of scale-out ranks");        int stored_scaleout_tail = 0, stored_old_scaleout_tail = 0;        const auto update_scaleout_tail = [&](const bool& finish_flag = false) {            if (lane_idx < kNumScaleoutRanks and                (stored_scaleout_tail >= stored_old_scaleout_tail + kScaleoutUpdateInterval or finish_flag)) {                // signaled_tail = (finish_flag << 32) | tail 值                const auto signaled_tail     = math::pack2<int, int64_t>(finish_flag, stored_scaleout_tail);                const auto ptr               = workspace_layout.get_scaleout_channel_signaled_tail_ptr(channel_idx, scaleout_rank_idx);                const auto old_signaled_tail = math::pack2<int, int64_t>(0, stored_old_scaleout_tail);                // red_add: 发送增量, 保证 forward warp 累加出最新 tail                gin.red_add_rel<ncclTeamTagRail>(ptr, signaled_tail - 13. Forward warp: 主循环 — 轮询多 peer signaled_tail
```
        // Forward tokens from scale-out ranks        // ★ 每 lane 跟踪一个 scaleout peer 的 (finish_flag, tail)；round-robin 挑选有数据的 peer        EP_STATIC_ASSERT(kNumScaleoutRanks <= 32, "Too many scale-out ranks");        int num_tokens_processed                           = 0;        int stored_scaleout_old_tail_idx                   = 0;        int stored_scaleup_send_counters[kNumScaleupRanksPerLane] = {};    // 每 scaleup peer 累计发送数        int stored_finish_flag                             = lane_idx >= kNumScaleoutRanks;  // 无效 lane 视为已完成        int stored_scaleout_tail_idx                       = 0;        int recv_scaleout_rank_idx = channel_idx % kNumScaleoutRanks;      // round-robin 起点        uint32_t wip_mask;        // wip_mask = 所有"有未处理 token 或尚未 finish"的 lane 的 bitmask；全为 0 时退出        while ((wip_mask = ptx::gather(                    stored_scaleout_tail_idx > stored_scaleout_old_tail_idx                    or stored_finish_flag == 0))) {            // Pick next rank in round-robin            // 按 (cur+1, cur+2, ..., 0, 1, ..., cur) 顺序挑第一个 wip_mask 置位的 lane            const auto offset  = (recv_scaleout_rank_idx + 1) % kNumScaleoutRanks;            const auto hi_mask = (wip_mask >> offset) << offset;            recv_scaleout_rank_idx = hi_mask ? ptx::ffs(hi_mask) : ptx::ffs(wip_mask);            // Wait for this rank to have data (or finish)            //   如果 peer 还没发新 tail, 读 signaled_tail 刷新寄存器, 再重试            comm::timeout_while<kNumTimeoutCycles>([&](const bool& is_last_check) {                const uint32_t arrived_or_finished =                    stored_scaleout_tail_idx > stored_scaleout_old_tail_idx or stored_finish_flag > 0;                if (ptx::exchange(arrived_or_finished, recv_scaleout_rank_idx))                    return true;                if (is_last_check) { /* printf */ return false; }                // Read new signaled tails                if (lane_idx < kNumScaleoutRanks) {                    const auto signaled_tail = ptx::ld_acquire_sys<int64_t>(                        workspace_layout.get_scaleout_channel_signaled_tail_ptr(channel_idx, lane_idx));                    // 拆包 (finish_flag, tail)                    math::unpack2<int, int64_t>(signaled_tail,                                                stored_finish_flag, stored_scaleout_tail_idx);                }                __syncwarp();                return false;            });            // Process one chunk from the current rank            // 本轮从 recv_scaleout_rank_idx peer 取 [start_slot, end_slot) 段, 最多 kNumSlotsPerForwardChunk 个            const auto start_slot_idx = ptx::exchange(stored_scaleout_old_tail_idx, recv_scaleout_rank_idx);            const auto end_slot_idx   = std::min(                ptx::exchange(stored_scaleout_tail_idx, recv_scaleout_rank_idx),                start_slot_idx + kNumSlotsPerForwardChunk);            if (lane_idx == recv_scaleout_rank_idx)                stored_scaleout_old_tail_idx = end_slot_idx;            const auto recv_buffer = scaleout_recv_buffer.get_rank_buffer(recv_scaleout_rank_idx);

15. Forward warp: 步骤 F3(linked-list 索引构建)
```
                // Write the per-scaleup channel index for this token                // ---- 步骤 F3: 为每个 topk 副本计算 channel_linked_list 索引, 写入 tma_buffer ----                // 关键技巧: stored_scaleup_send_counters[j] 保存 "本 warp 已向第 (j*32+lane) 个 scaleup peer 发过几个"                //   即每个 scaleup peer 独立一个计数器(跨 kNumScaleupRanksPerLane 个 warp-wide 条)                //   通过 ptx::exchange 把当前 token 要发给的那个 peer 的计数器值赋给 linked_list_idx,                 //   然后累加该 peer 的计数器 +1(在后面的 scaleup_send_mask 累加步骤)                int linked_list_idx = -1;#pragma unroll                for (int j = 0; j < kNumScaleupRanksPerLane; ++j) {                    const auto src_lane_idx = stored_dst_scaleup_rank_idx - j * 32;                    const bool valid = 0 <= src_lane_idx and src_lane_idx < 32;                    // 从对应 lane 的 counter 取当前值作为 linked_list_idx                    const auto exchanged = ptx::exchange(                        stored_scaleup_send_counters[j], valid ? src_lane_idx : 0);                    linked_list_idx = valid ? exchanged : linked_list_idx;                }                // 非 reuse 模式下把全局 linked list 下标写入 slot 的 linked_list_idx 字段                //   (kReuseSlotIndices 模式下 epilogue 不再构建链表, 无需写)                if (not kReuseSlotIndices and lane_idx < kNumTopk) {                    tma_buffer.get_linked_list_idx_ptr()[lane_idx] = transform_linked_list_idx(linked_list_idx);                    ptx::tma_store_fence();                }                __syncwarp();

```

```
18. Forward warp: 收尾 — metadata 终止符 + linked list 尾部 + tail 清零
```

```
19. Kernel 结尾: 仅 scale-up 维 barrier + PDL + 清零
```

```

Scale-up 角色流程（NVLink 上行）
```
while (true):  ① 并行读 kNumScaleupRanks 条 channel_linked_list 节点（每 lane 管 ceil(ranks/32) 个 peer）  ② 若所有链表到尾 → break  ③ 构造 wip_mask（本轮哪些 peer 有 token）  ④ Round-robin pop 目标 peer dst_scaleup_rank_idx：     ├── S1: 读 src_metadata 还原 (src_scaleout, src_token, src_slot)     │        → 构造 peer.scaleup_buffer 对称 VA token_buffer     ├── S2: 三分支装配 tma_buffer（与 Direct Combine 一致）     │   A) no_local_reduce: TMA load x[i]     │   B) local reduce: compute_topk_slots + combine_reduce → SMEM     │   C) expanded send: 每 topk 副本分别 store 到 peer.scaleup_buffer[k]     └── S3: 写 topk_weights, TMA store 到 peer.scaleup_buffer             stored_num_tokens_sent[对应lane]++  ⑤ update_tails()：每 3 轮批量 st_release_sys 推 tail 到所有 peerupdate_tails(finish=true)  // 最终 flush
```
Forward 角色流程（RDMA 下行）
```
for i = 0.. :  ① F1: 读 token_metadata_at_forward[i]（global_token, is_last, topk_scaleup[k], topk_slot[k]）      若哨兵 < 0 → break      构造 scaleup_mask（本 token 需要哪些 scaleup peer 的副本）  ② F2: timeout_while 等 tail 到达      ld_acquire_sys(channel_scaleup_tail[peer]) > stored_num_tokens_recv      每轮消费 +1  ③ F3 分支:     F3a 非 multi_reduce:        for k in topk_valid_mask:          TMA load peer.scaleup_buffer[k][src_slot]          TMA store → scaleout_send_buffer[k][i]（或本 scaleout bypass=recv_buffer）          gin.put<Rail>(src_scaleout, 聚合 doorbell, 最后一笔 flush)     F3b multi_reduce:        ptx::deduplicate 去重 + combine_reduce 多副本规约        [核心重叠点] combine_reduce 的 "wait release" 回调                      中插入 flush_last_tma_and_issue_rdma()  ← compute-comm overlap        TMA store → send_buffer，记录 last_* 下一轮 flush  ④ F4: 清零 channel_scaleup_tail（为下次 dispatch 复用）  ⑤ F5: red_add_rel<Rail> → 所有 scaleout peer 的 signaled_tail         timeout_while 等所有 peer 的 red_add → 清零// 无 final barrier：red_add+wait 已替代
```
