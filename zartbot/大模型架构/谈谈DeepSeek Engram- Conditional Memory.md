## 1. Overview

### 1.1 核心问题

Transformer 架构缺乏一种原生的知识查找原语, 迫使其通过**计算来低效地模拟检索**.

![图片](assets/50c8a32e1659.jpg)

原文摘要中"Transformers lack a native primitive for knowledge lookup"写的挺准确的, Transformer用统一的, 计算密集的自注意力 (self-attention) 和 FFN 来处理所有任务构建序列模型. 但是Next-token-Prediction这样的自回归"一刀切"的方法在处理**静态, 模式化知识** (如固定短语, 命名实体) 时是极其低效的.

"inefficiently simulate retrieval through computation"描述了这种现状, 模型为了回忆起一个地名的实体, 例如"New York", 可能需要好几层网络进行特征提取和组合, 这本质上是在用昂贵的动态计算去模拟一个本该是廉价的静态查找操作. 从计算机体系结构的角度看, 这无疑是非常浪费的.

### 1.2 "计算" vs "检索"

作者将 Transformer 需要处理的信息明确划分为两类

**组合推理 (Compositional Reasoning)**: 需要深入、动态的计算, 适合由 MoE 这样的计算密集型模块处理.

**知识检索 (Knowledge Retrieval)**: 涉及大量静态、固定的模式 (如命名实体、常用短语), 这些模式本质上更适合通过高效的查找操作来解决.

从算法的角度来看, 则可以分为如下两类:

**动态计算**: 指的是需要根据全局上下文进行复杂推理、组合和消歧的任务, 这些任务是需计算的. 例如:

代词指代: "He said..." 中的 "He" 指的是谁?

逻辑推理: "A is greater than B, B is greater than C, so A is..."

情感分析: "This movie was anything but boring." 这句话是正面还是负面评价?

**静态模式检索**: 指的是那些不随上下文改变其核心含义的、固定的语言片段,这些模式是可记忆的. 例如:

命名实体: "New York","Albert Einstein"

固定短语/习语: "by the way", "state-of-the-art",

专业术语: "gradient descent", "machine learning"

因此在网络设计上, 采用了一个专门的、廉价的**存储模块 (Engram)** 来处理静态模式检索. 而让昂贵的**计算模块 (Transformer Block)** 专注于动态计算.

引入Engram后, 原文有一句"effectively deepening the network": 我们可以这么理解, 模型的物理深度没有改变, 但由于它跳过了低级任务, 直接从一个更高级的语义起点开始处理信息, 其"有效深度"得到了增加. 更深的网络通常意味着更强的推理能力, 这就解释了为何推理任务受益最大.

### 1.3 从稀疏性的视角来看

由于作者将待处理的信息分为静态模式匹配和动态计算两类, 那么从稀疏性的视角来看, 作者给出了两个正交的稀疏性方向:

**条件计算**: 针对动态计算过程, MoE 通过条件计算来扩展模型容量, 针对每个Token, 通过路由器选择稀疏性的激活少数几个专家. 本质上它是一个动态的计算过程, 并且很好的匹配了上下文的逻辑依赖和一些复杂的任务.

**条件记忆**: 针对静态模式模式匹配, 依赖稀疏的查找操作来检索静态知识的嵌入, 例如可以将这些静态的知识储存起来, 类似于外挂一个词典, 通过快速查表的方式直接检索知识向量.

### 1.4 具体工程实现

在具体工程实现的过程中需要考虑如下几个问题:
1. 如何针对超大规模的静态表进行快速检索?
作者采用了N-gram算法, 然后通过 Hash 将近乎无限的 N-gram 空间映射到有限的内存地址上, 实现了 $O(1)$ 的查找效率.
2. 对于静态模式检索的结果如何融合回骨干的Transformer网络?
检索到的知识是**静态的、上下文无关的**, 而模型所处的环境是**动态的、上下文相关的**. 因此需要一个"融合(Fusion)"算法解决这个矛盾.
3. 对于"计算"和"检索"的资源分配如何处理?
也就是前一节所讲的, 作者给出了两个正交的稀疏性方向:**条件计算**来获得计算的稀疏性,**条件记忆**来获得访问内存的稀疏性, 但是它们应该如何在整体的计算资源中分配? 作者通过构建"稀疏性分配" (Sparsity Allocation) 问题, 发现了一条 U 型缩放定律

![图片](assets/2db5ca6e1a84.jpg)
4. 基础设施如何协同?
Engram 建立了"基础设施感知" (infrastructure-aware) 的效率: 其确定性的寻址逻辑允许在运行时从主机内存 (host memory) 进行预取 (prefetching), 带来的开销可以忽略不计.

## 2. Engram架构

### 2.1 Overview

如图所示, Engram 是一个条件内存模块, 其设计目的是通过在结构上将**静态模式存储**与**动态计算**分离开来, 从而增强 Transformer 的骨干网络.

![图片](assets/ca4f9e94ca8d.png)

形式上, 给定一个输入序列 $X = (x_1, \dots, x_T)$ 以及在第 $\ell$ 层的隐藏状态 $H^{(\ell)} \in \mathbb{R}^{T \times d}$, 该模块对每个位置 $t$ 的处理都包含两个功能阶段: **检索 (retrieval)** 和 **融合 (fusion)**. 这个两阶段流程清晰地定义了 Engram 的工作方式.

检索 (Retrieval):

**"deterministically retrieve" (确定性地检索)**: 这是 Engram 与 MoE 的一个关键区别. MoE 的专家选择是**动态的**, 依赖于运行时的隐藏状态. 而 Engram 的检索目标 (N-gram) 是**确定性的**, 仅由输入文本决定. 这个特性是第 2.5 节系统优化的基石. 在模型开始计算某一层之前, 系统就已经知道了所有 Engram 层需要哪些内存条目, 因此可以做一些Prefetch的处理

**"via hashing" (通过哈希)**: 这揭示了实现大规模检索的核心技术.

融合 (Fusion):

**"dynamically modulated by the current hidden state" (被当前隐藏状态动态地调制)**: 这点出了"融合"阶段的智能所在. 检索到的知识是**静态的、上下文无关的**, 而模型所处的环境是**动态的、上下文相关的**. "融合"阶段的核心任务就是解决这个矛盾. 它使用当前的隐藏状态 $h_t$ (它已经编码了全局上下文信息) 去"审查"和"调制"检索到的静态知识, 判断它是否适用、如何适用, 从而实现动态与静态的智能结合. 这就是 2.3 节 "上下文感知门控" 的作用.

在Overview这一节, 作者给出了一个很清晰的文档结构:

**2.2 节**: 讲如何"查" (Retrieval).

**2.3 节**: 讲如何"用" (Fusion).

**2.4 节**: 讲如何与特定的高级架构"适配".

**2.5 节**: 讲如何在真实硬件上"高效运行".

### 2.2 稀疏检索: 基于哈希的 N-gram

这一节是 Engram 模块的核心, 描述了如何从输入的 token 序列中高效、确定性地提取与局部上下文相关的静态知识. 这个过程可以看作是 Engram 的检索 (Retrieval) 阶段.这包括**分词器压缩**和通过**确定性哈希**检索嵌入两个步骤.

#### 2.2.1 分词器压缩 (Tokenizer Compression)
需要解决的问题: 词汇冗余和语义离散
标准的Subword分词器的首要目标是无损地重构原始文本. 这个特性导致了一个问题: 语义上完全相同或高度相似的词, 仅仅因为表面形式的微小差异 (如大小写, 前导空格), 就可能被赋予完全不同的 token ID.下面是一个例子:

论文中给出的 `Apple` 和 `␣apple` (带前导空格的'apple'), 以及类似的 `run` vs `running`, `USA` vs `U.S.A.`, 它们在很多上下文中指向同一个概念, 但在分词器看来是不同的"原子单位".

这种"一义多词"的现象导致了:

**参数效率低下**: 模型需要为这些语义相近的 token 分别学习嵌入, 浪费了参数.

**数据稀疏**: 模型在文本中见到 `Apple` 的次数和见到 `␣apple` 的次数是分开计数的, 这使得学习它们的共同语义变得更加困难.

**N-gram 空间爆炸**: 对于 Engram 这种依赖 N-gram 的模块, 这种冗余会不必要地大大增加 N-gram 组合的数量.
解决方案: 词汇投影及规范化
Engram 采用了一种简单而高效的预处理方法来解决这个问题.

首先, 定义一个满射函数 $P: V \to V'$:

$V$ 是原始的、庞大的词汇表 (例如, 128k 个 token).

$V'$ 是一个更小的、语义更密集的"规范化"词汇表.

$P$ 是一个**多对一**的映射, 它将多个原始 token ID "折叠"到一个单一的规范 ID 上.

具体实现方式: 这个映射是基于**文本等价性**的. 具体来说, 它会对每个 token 对应的文本进行一系列规范化操作, 例如:

NFKC 规范化: 一种 Unicode 规范化形式, 可以统一全角/半角字符、不同的符号变体等.

小写转换: 将 `Apple` 和 `apple` 统一为 `apple`.

去除特殊符号或前导空格.

所有经过规范化后文本相同的原始 token, 都会被映射到同一个规范 ID.
形式化的描述
对于位置 $t$ 的一个 token, 我们将其原始 ID $x_t$ 映射为一个规范 ID $x_t' = P(x_t)$, 以此来形成 Suffix N-gram $g_{t,n} = (x_{t-n+1}', \dots, x_t')$.
效果
**有效缩小词汇表**: 论文提到, 对一个 128k 的词汇表, 这个过程能实现 23% 的缩减. 这意味着 Engram 的 N-gram 是在约 98k 的有效词汇表上构建的, 极大地降低了组合复杂性.

**提升语义密度**: 经过压缩后, 一个规范 ID 背后聚集了多个原始 token 的统计信息, 其对应的嵌入向量能够更稳健地学习到核心语义, 不再受到表面形式的干扰.

这是对 Engram 的一个关键优化: 如果没有这一步, Engram 的内存表将需要为大量冗余的 N-gram 分配空间, 导致容量被浪费, 性能下降 (这在 6.2 节的消融实验中得到了验证).

#### 2.2.2 多头哈希 (Multi-Head Hashing)
需要解决的问题: N-gram 组合爆炸
即使经过了分词器压缩, N-gram 的潜在组合数量仍然是天文数字.例如假设有效词汇表大小 $|V'| \approx 10^5$.

2-gram 的数量级是 $(10^5)^2 = 10^{10}$ (百亿).

3-gram 的数量级是 $(10^5)^3 = 10^{15}$ (千万亿).

为每一种可能的 N-gram 分配一个独立的、可学习的嵌入向量是完全不可行的, 内存开销无法承受. 这就是 N-gram 模型需要面对的**数据稀疏性**和**维度灾难**问题.
解决方案: Hashing Trick
其核心思想是, 不再为每个 N-gram 分配唯一 ID, 而是通过一个Hash函数将其映射到一个固定大小的Hash桶中的某个位置, 但是如果使用一个单一的Hash函数会导致潜在的Hash冲突. 因此Engram设计了一个多头Hash(Multi-head Hasing)算法, 为每个 N-gram 阶数 $n$ 设计了 $K$ 个**独立**的哈希头. 每个头 $k$ 包含:

一个独立的确定性哈希函数 $\phi_{n,k}$.

一个独立的嵌入表 $E_{n,k}$ (其大小 $M_{n,k}$ 为素数)

![图片](assets/cd591cacaa5c.jpg)

这描述了单个哈希头的操作: 将 N-gram $g_{t,n}$ 通过哈希函数 $\phi_{n,k}$ 得到索引 $z_{t,n,k}$, 然后用该索引从对应的嵌入表 $E_{n,k}$ 中取出嵌入向量 $e_{t,n,k}$.

$$z_{t,n,k} = \phi_{n,k}(g_{t,n}), \quad e_{t,n,k} = E_{n,k}[z_{t,n,k}] \tag{1}$$

在具体实现中, $\phi_{n,k}$ 被实现为一个轻量级的XOR Hash, 并通过拼接所有检索到的嵌入来构建最终的内存向量 $e_t$, 其中 $\Vert$ 表示拼接操作

$$e_t \triangleq \mathop{\big\Vert}_{n=2}^{N} \mathop{\big\Vert}_{k=1}^{K} e_{t,n,k} \tag{2}$$

这描述了最终内存向量的构建过程. 整个过程是:

对每个 N-gram 阶数 $n$ (例如, 2-gram, 3-gram), 分别计算 $K$ 个头的嵌入.

将所有阶数、所有头的嵌入向量全部拼接起来, 形成一个长的、复合的内存向量 $e_t$.

这个 $e_t$ 向量就作为对当前位置 $t$ 局部上下文的静态知识编码, 传递给下一阶段的融合模块.

渣注:
这是一个非常简单有效的避免Hash冲突的方式, 两个不相关的 N-gram 同时在所有 $K$ 个哈希头上都发生冲突的概率非常低 (如果哈希函数设计良好). 即使在一两个头上发生冲突, 其他头检索到的"干净"嵌入向量依然能提供有效信息. 另一方面我们注意到嵌入表 $E_{n,k}$ 大小为素数, 将哈希表的大小设为素数是一种标准技巧, 有助于减少因输入数据的规律性而导致的系统性冲突, 使哈希值分布更均匀. 并且在计算上使用了最简单的XOR满足高效的运算效率.

另一方面, 即便是有Hash冲突, 例如冲突后产生的噪音在下一节的门控机制 能够利用当前隐藏状态 $h_t$ 的上下文信息来判断检索出的内存向量 $e_t$ 是否与当前语境相关. 如果 $e_t$ 因为冲突而包含了不相关的信息, 门控值 $\alpha_t$ 就会变小, 从而抑制这个噪声向量的影响.

### 2.3 上下文感知门控(Context-aware Gating)

这一节是 Engram 模块的第二阶段, 即**融合 (Fusion)** 阶段. 它描述了如何将 2.2 节中检索到的静态内存向量 $e_t$ 与模型当前的动态隐藏状态 $h_t$ 进行智能地结合. 这个门控机制是 Engram 能够超越简单 N-gram 嵌入, 并有效融入复杂 Transformer 架构的关键. 核心的想法是: 不盲目地信任从内存中检索出的信息, 而是让模型根据当前的全局上下文来动态地决定是否使用以及如何使用这些信息. 这个过程可以分解为三个步骤: **QKV 角色分配与投影**, **门控计算**, 以及 **后处理与融合**.
1. 门控的动机
从 2.2 节得到的内存向量 $e_t$ 是上下文无关的. 它的值完全由其后缀 N-gram 决定, 而不考虑句子中其他部分的内容. 这带来了两个主要问题:

**歧义性 (Ambiguity / Polysemy)**: 同一个 N-gram 在不同上下文中可能有完全不同的含义. 例如, "apple" 这个 N-gram, 在 "I ate an apple" 中指水果, 在 "I bought an Apple computer" 中指公司. 静态的 $e_t$ 无法区分这两种情况.

**噪声 (Noise)**: 由于哈希冲突, 检索出的 $e_t$ 可能混合了不相关 N-gram 的信息. 即使没有冲突, 对于某些罕见或无意义的 N-gram, $e_t$ 本身也可能是无用的噪声.

为了解决这些问题, Engram 需要一个机制来判断 $e_t$ 在**当前上下文**中是否适用. 这个机制就是**上下文感知门控**.
2. 门控机制借鉴了Attention的处理
Engram 的门控机制巧妙地借用了标准 Attention 模块的 Query-Key-Value (QKV) 思想, 但做了简化和特化.

![图片](assets/3e029688717c.png)

**Query (Q)**: 当前 Transformer 层的隐藏状态 $h_t$. 这是门控机制中 "上下文感知" 的来源. $h_t$ 已经通过了之前所有层的注意力计算, 聚合了整个序列的全局上下文信息. 因此, $h_t$ "知道" 当前位置 $t$ 所在的语境是什么.

**Key (K) & Value (V)**: 检索到的静态内存向量 $e_t$. 与标准自注意力 (self-attention) 中 Q, K, V 都来自同一个源 ($h_t$) 不同, 这里的 K 和 V 来自外部的内存 $e_t$.

$k_t = W_K e_t$: 将 $e_t$ 投影到 "Key" 空间. 这个 $k_t$ 代表了 $e_t$ 所包含的静态知识的 "内容摘要".

$v_t = W_V e_t$: 将 $e_t$ 投影到 "Value" 空间. 这个 $v_t$ 代表了如果这份知识被采纳, 应该向模型注入的 "信息内容".

门控计算的公式如下:

$$\alpha_t = \sigma\left(\frac{\text{RMSNorm}(h_t)^\top \text{RMSNorm}(k_t)}{\sqrt{d}}\right) \tag{4}$$

这是一个标量门控, 其计算过程几乎就是一个简化的点积注意力:

**对齐判断**: 计算 $h_t$ (Query) 和 $k_t$ (Key) 的点积. 这个点积衡量了当前上下文 ($h_t$) 与检索到的静态知识 ($k_t$) 之间的语义对齐程度 (semantic alignment).

如果 $h_t$ 和 $k_t$ 语义上高度相关 (例如, 语境是 "水果", 检索到的 N-gram 是 "red apple"), 它们的投影向量方向会很接近, 点积结果会很大.

如果 $h_t$ 和 $k_t$ 语义上不相关或矛盾 (例如, 语境是 "科技公司", 检索到的 N-gram 因为哈希冲突包含了 "of the" 的信息), 点积结果会很小甚至为负.

**归一化 (RMSNorm)**: 在计算点积前对 Query 和 Key 进行 RMSNorm. 这是现代 LLM 架构中为了保证训练稳定性的常用技巧, 可以防止向量范数过大或过小导致点积结果不稳定, 从而使梯度更加平滑.

**缩放 ($1/\sqrt{d}$)**: 经典的 Attention 缩放因子, 防止点积结果过大, 导致 Sigmoid 函数进入饱和区, 造成梯度消失.

**激活函数 ($\sigma$)**: 使用 Sigmoid 函数将点积结果压缩到 $[0,1]$ 区间, $\alpha_t$ 可以被直观地理解为 "当前上下文对检索到的这份知识的信任度或采纳度".

最后通过 $o_t = \alpha_t \cdot v_t$ 将计算出的门控值 $\alpha_t$ 逐元素乘以 Value 向量 $v_t$.

如果 $\alpha_t \to 1$, 表示上下文与内存高度匹配, 检索到的信息 $v_t$ 将被几乎完整地保留.

如果 $\alpha_t \to 0$, 表示上下文与内存不匹配, 检索到的信息 $v_t$ 将被完全抑制, 避免了噪声或歧义信息的污染.
3. 后处理: 深度因果卷积 (Depthwise Causal Convolution)
门控后的输出序列 $o_t$ 仍然是逐个 token 独立处理的. 为了在相邻的 token 之间建立联系, 并进一步增强模型的表达能力, Engram 引入了一个轻量级的卷积层.

$$Y = \text{SiLU}(\text{Conv1D}(\text{RMSNorm}(\hat{V}))) + \hat{V} \tag{5}$$

通过一个卷积层有如下好处:

**扩大感受野**: 卷积核可以在时间维度上整合邻近 token 的信息. 一个大小为 $w$ 的卷积核可以让每个位置的输出感知到其前面 $w-1$ 个位置的信息, 从而对局部模式进行更复杂的建模.

**增强非线性**: 配合 SiLU 激活函数, 卷积层为模块增加了额外的非线性计算能力.

具体实现采用了depthwise causal convolution:

**深度卷积 (Depthwise Convolution)**: 每个输入通道 (特征维度) 由一个独立的卷积核处理. 相比标准卷积, 它的参数量和计算量都小得多, 符合 Engram 轻量级设计的原则.

**因果卷积 (Causal Convolution)**: 通过 padding 确保在计算位置 $t$ 的输出时, 只会用到位置 $t$ 及其之前的信息, 保持了自回归模型的因果性.

参数设置采用:

`kernel_size=4`: 一个很小的卷积核, 只关注非常局部的交互.

`dilation=max N-gram order`: 可以在不增加计算量的情况下扩大感受野. 将其设置为最大 N-gram 阶数, 可能旨在让卷积的感知范围与 N-gram 的范围相匹配?

另外从公式(5)中可以看到它还使用了残差连接: $Y = \text{SiLU}(\text{Conv1D}(\text{RMSNorm}(\hat{V}))) + \hat{V}$ 使用了残差连接. 这确保了即使卷积层没有学到任何有用信息, 原始的门控输出 $\hat{V}$ 也能无损地通过, 保证了训练的稳定性.
4. 最终融合与放置策略
最终的输出 $Y$ 通过残差连接加到主干网络的隐藏状态上: $H^{(\ell)} \leftarrow H^{(\ell)} + Y$. 在放置测量上, 论文特别强调, Engram **不是**应用在每一层. 其具体放置位置 (Layer 2, 15 等) 是一个需要权衡建模效果和系统延迟的协同设计问题, 这在 6.2 节有详细讨论.

渣注
这一节是整个方案中实现非常巧妙的一环. 通过类似于Attention的机制, 当前全局上下文 ($h_t$) 作为 Query, 检索到的静态知识 ($e_t$) 作为 Key/Value, 实现了一个动态的、上下文感知的信息过滤和融合机制,有效地处理了静态内存固有的**歧义性**和**噪声**问题.

另一方面, 整个门控和后处理过程都由轻量级操作构成, 保持了 Engram 模块低计算开销的特性.

### 2.4 整合到Hyper-Connection

在这里和DeepSeek mHC联系起来了, 在经典的 ResNet 或 Transformer 中, 信息沿着一个单一的残差流 (residual stream) 传递. 每个块的输出都是在前一个块输出的基础上进行修改.

而前面我们已经介绍过[《谈谈DeepSeek mHC》](https://mp.weixin.qq.com/s?__biz=MzUxNzQ5MTExNw==&mid=2247497138&idx=1&sn=8215a15d8e196d412ab908ec3302c857&scene=21#wechat_redirect), 实际上将残差流从标准单流架构 (Single-stream)扩展成了  个并行的分支.当要把 Engram 模块集成到这样的多分支架构中时, 出现了一个设计选择:

**简单方案**: 在所有分支聚合之前或之后, 应用一个统一的 Engram 模块. 这种方法简单, 但可能无法充分利用多分支架构的优势.

**复杂方案**: 为每个分支配备一个完全独立的 Engram 模块. 这种方法表达能力最强, 但参数量和计算开销会增加 $M$ 倍, 违背了 Engram 轻量级的设计初衷.

Engram 的设计目标是在**效率** 和**表达能力** 之间取得最佳平衡. 论文提出了一种非常巧妙的**半共享 (semi-shared)** 策略:
完全共享的部分 (Shared Components):
**稀疏嵌入表 (Sparse embedding table)**: 所有 N-gram 的嵌入表是全局共享的. 这部分是 Engram 参数量的大头, 共享它可以极大地节约内存.

**Value 投影矩阵 $W_V$**: 所有分支共享同一个 $W_V$. 这意味着从静态内存 $e_t$ 中提取的 "信息内容" ($v_t$) 对所有分支来说是统一的.
分支特化的部分 (Branch-specific Components):
**Key 投影矩阵 $W_K^{(m)}$**: 每个分支 $m$ 拥有一个**独立**的 Key 投影矩阵 $W_K^{(m)}$.分支特化的部分计算如下:

$$\alpha_t^{(m)} = \sigma\left(\frac{\text{RMSNorm}(h_t^{(m)})^\top \text{RMSNorm}(W_K^{(m)} e_t)}{\sqrt{d}}\right) \tag{6}$$

渣注
这样的做法让不同的分支可能关注不同的语义信息. 例如, 一个分支可能关注语法结构, 另一个分支可能关注实体关系. 让每个分支拥有自己的 Key 投影, 意味着每个分支可以学习一套自己的 "标准" 来判断检索到的静态知识 $e_t$ 是否与自己所关注的语义维度相关. 具体来看:

对于同一个检索到的内存向量 $e_t$, 每个分支 $m$ 会计算出一个分支特定的门控值 $\alpha_t^{(m)}$.

$\alpha_t^{(m)}$ 反映了 $e_t$ 与第 $m$ 个分支的隐藏状态 $h_t^{(m)}$ 的相关性.

最终, 所有分支都会作用于同一个信息内容 $v_t = W_V e_t$, 但每个分支会根据自己的判断 ($\alpha_t^{(m)}$) 来决定 "吸收" 多少信息:

$$h_t^{(m)} \leftarrow h_t^{(m)} + \alpha_t^{(m)} \cdot (W_V e_t)$$

这种设计既允许了分支行为的多样性和特化, 又通过共享大部分参数来保持了高效率.

另一方面, 作者顺带提了一下算子融合的优化, 表面上看, 计算 $M$ 个独立的 Key 投影和 1 个共享的 Value 投影需要 $M+1$ 次矩阵乘法.

$$k_t^{(1)} = W_K^{(1)} e_t, \quad k_t^{(2)} = W_K^{(2)} e_t, \quad \dots, \quad k_t^{(M)} = W_K^{(M)} e_t, \quad v_t = W_V e_t$$

将上述 $M+1$ 个独立的投影操作融合成了一次计算, 即将所有 Key 投影矩阵 $W_K^{(m)}$ 和共享的 Value 投影矩阵 $W_V$ 在维度上拼接起来, 形成一个更大的、统一的投影矩阵 $W_{\text{fused}}$:

$$
W_{\text{fused}} = \begin{bmatrix} W_K^{(1)} \\
W_K^{(2)} \\
\vdots \\
W_K^{(M)} \\
W_V \end{bmatrix}
$$

然后, 只需进行一次矩阵乘法:

$$[k_t^{(1)}; k_t^{(2)}; \dots; k_t^{(M)}; v_t] = W_{\text{fused}} \cdot e_t$$

最后, 将得到的长向量 $W_{\text{fused}} \cdot e_t$ 切分回各个分支所需的 $k_t^{(m)}$ 和共享的 $v_t$.

### 2.5 系统效率: 解耦计算与内存

Engram 的**确定性检索 (deterministic retrieval)** 特性, 是其能够实现高效、可扩展系统部署的关键.首先作者阐述了MoE的区别以及为什么具有确定性检索能力:
MoE
**依赖运行时状态**: MoE 中的路由决策 (即一个 token 应该被发送到哪个专家) 是由一个门控网络 (gating network) 做出的, 而这个门控网络的输入是当前层的隐藏状态 $h_t$.

**不可预测性**: 在开始计算第 $\ell$ 层之前, 你无法知道哪些专家会被激活. 必须先计算完前 $\ell-1$ 层得到 $h_t$, 才能进行路由, 然后才能加载或激活相应的专家参数.

因此计算和通信/内存访问是**串行**的. 路由决策 -> 加载专家 -> 执行计算. 这种模式使得将专家参数卸载到主机内存或 SSD 变得非常困难, 因为等待数据传输的延迟会直接导致 GPU stall, 严重影响吞吐量.
Engram (确定性检索):
**仅依赖输入序列**: Engram 的检索索引 (哈希值) 完全由**输入的 token 序列**决定.

**可预测性**: 只要拿到了完整的输入文本, 在模型开始进行任何前向计算**之前**, 就可以计算出每一层 Engram 模块需要访问的所有 N-gram 的哈希索引.

因此计算和内存访问可以**解耦 (decouple)** 并**并行 (overlap)** 执行. 这是一个巨大的系统优势.

然后整个系统设计如下:

![图片](assets/519c74001ca9.png)
训练阶段的优化: 模型并行
在训练阶段, Engram 的巨大嵌入表无法装入单个 GPU. 论文采用了标准的模型并行策略, 即分片(sharding).将巨大的嵌入表按行切分, 分散到所有参与训练的 GPU 上.

**前向传播**: 当需要某个嵌入向量时, 如果它不在本地 GPU, 就通过网络通信从持有它的 GPU 那里获取. `All-to-All` 是一种高效的集合通信原语, 可以让每个 GPU 同时向所有其他 GPU 发送自己需要的数据请求, 并接收自己持有的数据被请求的部分.

**反向传播**: 梯度也需要通过类似的方式 (`All-to-All`) 分发回持有相应参数的 GPU 进行更新.

这种方法使得 Engram 的总内存容量可以随着加速器数量的增加而线性扩展.对于一个超大规模的训练集群应该可以很容易放下这些Embedding Table.
推理阶段的优化: Prefetch & Overlap
由于确定性检索和在推理集群上通常整个集群GPU内存容量不太大,可以借助CPU的内存资源, 并利用CPU进行Hash Lookup.具体流程如下:

在推理开始时, 首先遍历输入 token 序列, 计算出所有将要被访问的 Engram 层的 N-gram 及其哈希索引. 当 GPU 开始计算模型的第 1 层、第 2 层...时, CPU 可以同时异步发起一个指令, 让数据从主机内存 (Host DRAM) 通过 PCIe 总线传输到 GPU 内存 (HBM) 中. 传输的内容就是稍后某个深层 Engram 模块将要用到的嵌入向量.

GPU 正在忙于计算前面几层 Transformer block 的 Attention 和 FFN, 这些计算耗费的时间正好可以用来"掩盖"数据通过 PCIe 传输所需的时间.当 GPU 计算到需要 Engram 模块的第 $\ell$ 层时, 从主机内存预取的数据已经到达了 GPU 内存, GPU 可以立即使用, 无需等待.
层次化存储
自然语言中的 N-gram 频率遵循Zipf's Law, 这意味着:

小部分高频 N-gram (如 "of the", "in the") 占据了绝大多数的访问次数.

大量的低频 N-gram (如 "quantum entanglement", "serendipitous discovery") 很少被访问.

将所有 N-gram 的嵌入都放在同一种存储介质上是低效的. 我们可以根据访问频率, 将它们分层存储, 这种分层存储策略使得 Engram 可以扩展到**万亿 (Trillion)** 级别的参数规模, 而其**有效访问延迟** 却只由少数高频条目的访问时间决定. 这在理论上为模型容量的无限扩展提供了可能, 而无需昂贵的硬件升级.

## 3. 稀疏性分配

作为条件化内存的一种实例化, Engram 在结构上与 MoE 专家提供的条件化计算是互补的. 因此需要回答两个资源分配相关的问题:

**有限约束下的分配.** 当总参数量和训练计算量都固定时 (即同等参数量和同等浮点运算次数), 我们应该如何在 MoE 专家和 Engram 嵌入之间划分稀疏容量?

**无限内存机制.** 考虑到 Engram 的开销是 O(1) 且不随规模增长, 如果内存预算被放宽或进行激进地扩展, Engram 自身会展现出怎样的缩放行为?

渣注
首先文章阐述了MoE和Engram时结构上互补(Structurally complementary)的.

**MoE (条件化计算)**: 擅长处理需要**动态、深度、组合式推理**的任务. 它的能力来源于在大量专业化的计算单元 (专家) 之间进行灵活的路由.

**Engram (条件化内存)**: 擅长处理**静态、模式化、可记忆**的知识. 它的能力来源于对一个巨大知识库的快速 () 查找.

但是需要研究有限预算下的"资源分配"问题, 在**Iso-parameter (同等参数)** 和 **Iso-FLOPs (同等计算量)** 约束下:

是全部投入到 MoE, 增加专家的数量, 提升模型的计算多样性?

还是全部投入到 Engram, 增加内存表的大小, 提升模型的知识容量?

还是以某种比例进行混合分配?

另一方面和研究相关, Engram 是其存储和计算的解耦. 理论上, 我们可以通过将 Engram 表卸载到主机内存或 SSD, 以极低的成本将内存容量扩展到非常大的规模.

### 3.1 MoE 与 Engram 之间的最优分配比例

这一节主要分析在固定的参数和计算预算下, 如何最优地在 MoE 和 Engram 之间分配稀疏容量. 作者首先定义了三个关键的参数指标:

$P_{\text{total}}$ (总参数): 衡量模型的**存储成本**.

$P_{\text{act}}$ (激活参数): 衡量模型的**计算成本 (FLOPs)**.

$P_{\text{sparse}}$ (稀疏参数): 衡量模型的 **"免费"扩容潜力** ($P_{\text{sparse}} = P_{\text{total}} - P_{\text{act}}$). 这是整个实验设计的核心概念.

实验的核心是保持 $P_{\text{total}}$ 和 $P_{\text{act}}$ 在同一计算预算下**恒定**. 这意味着所有参与比较的模型:

**计算成本完全相同**: 每次前向传播的 FLOPs 一样, 训练相同 token 数所需的总计算量也一样.

**总存储成本也几乎相同**: 模型的总大小一样.

然后引入一个分配变量 $\rho \in [0,1]$.其值覆盖了从纯 Engram 模型 ($\rho=0$) 到纯 MoE 模型 ($\rho=1$) 的整个谱系.

$$P_{\text{MoE}}^{(\text{sparse})} = \rho P_{\text{sparse}}, \quad P_{\text{Engram}} = (1-\rho) P_{\text{sparse}} \tag{7}$$

通过调整 $\rho$, 作者可以精确地控制模型架构: 减少一个 MoE 专家, 就将这部分参数预算转移给 Engram 的嵌入表, 反之亦然.
实验设置:
实验设置在两个计算规模 (Regimes)约束下, 即两个不同的计算预算 ($2 \times 10^{20}$ 和 $6 \times 10^{20}$ FLOPs) 下进行, 并以恒定的稀疏比: $P_{\text{total}}/P_{\text{act}} \approx 10$. 这意味着总参数量大约是激活参数量的 10 倍, 这是一个比较典型的稀疏模型设置. 只有 MoE 专家的数量和 Engram Embedding Slot的数量在变, 其他所有超参数 (如优化器、学习率、数据等) 完全一致.

$C = 2 \times 10^{20}$ FLOPs: $P_{\text{act}} = 5.7B$ and $P_{\text{sparse}} = 568M$. The baseline ($\rho=1$) has a total of 106 experts.

$C = 6 \times 10^{20}$ FLOPs: $P_{\text{act}} = 9.9B$ and $P_{\text{sparse}} = 993M$. The baseline ($\rho=1$) has a total of 99 experts.
实验结果:
![图片](assets/1f27fd632101.png)

如上图所示, Validation Loss与分配比例 $\rho$ 之间呈现出一条清晰的 **U 形曲线**. 这是本节最重要的发现. 这条曲线告诉我们:

**两端非最优**: 曲线的两个端点, 即纯 MoE ($\rho=1$) 和纯 Engram (或 Engram 主导, $\rho \to 0$), 都不是性能最好的点.

**混合即是更优**: 曲线的最低点出现在中间区域, 意味着将稀疏预算**混合分配**给 MoE 和 Engram, 可以获得比任何单一策略都更好的性能.

**最优区间的存在**: 在两个不同的计算规模下, 曲线的最低点都稳定出现在 $\rho \in [0.75, 0.8]$ 的范围内. 这意味着将大约 20%-25% 的稀疏预算分配给 Engram, 剩下的 75%-80% 分配给 MoE, 是一个鲁棒的最优策略.

对于U形曲线:

纯 MoE 模型缺乏专门的硬件来"记忆"静态模式 (如固定短语、命名实体). 它必须浪费宝贵的计算资源 (FFN 层) 来"模拟"或"重构"这些本可以被简单查找的模式. 引入 Engram 后, 模型可以将这部分低级任务外包出去, 从而将 MoE 的计算能力解放出来, 专注于更高级的、动态的推理任务. 这种分工合作提升了整体效率.

另一方面内存不能完全替代计算. 语言理解不仅仅是模式匹配和知识检索, 还包含大量的组合推理、逻辑推导和上下文理解, 这些任务需要动态的、深度的计算. 过度削减 MoE 的容量会损害模型处理这些复杂任务的能力, 导致性能下降.

另一方面还有一个比较有趣的结论, 将纯 MoE 模型中60%的专家砍掉, 把这些参数换成 Engram 内存, 得到的混合模型性能居然和原来的纯 MoE 模型差不多. 这充分说明了 Engram 在参数效率上的优势.

### 3.2 Engram 对数线性缩放定律

使用一个固定的 MoE 网络, 其总参数 $P_{\text{total}} = 3B$, 激活参数 $P_{\text{act}} = 568M$, 并训练 100B tokens 以确保收敛. 在这个网络之上, 附加一个 Engram 表, 并将Embedding Slots的数量 $M$ 从 $2.5 \times 10^7$ 到 $1.0 \times 10^9$ (这额外增加了约 130 亿的参数). 作为基线, 与 OverEncoding 方法进行比较.

这是一个非常合适的基线. OverEncoding 也是一种利用 N-gram 嵌入来增强模型的方法, 但它的融合方式比较简单: 将 N-gram 嵌入与 token 的常规词嵌入直接平均. 通过对比 Engram 和 OverEncoding, 实验可以回答: "性能提升, 究竟是因为用了 N-gram 内存, 还是因为用了一种更智能的融合方式 (上下文门控)?"

![图片](assets/3f8b667ef835.png)

上图结果揭示了两个关键信息:

**内存越大, 性能越好**: 随着 Engram Embedding Slots数量 $M$ 的增加, Validation Loss持续稳定地下降. 这证明了为模型提供更多的静态知识储备是有益的.

**可预测的幂律关系**: 图表的横轴是嵌入槽数量 (对数坐标), 纵轴是验证损失 (线性坐标). 在这个 log-linear 坐标系中, Engram 的性能曲线呈现出一条近乎完美的直线.这意味着损失 $L$ 与内存Slots数量 $M$ 之间存在一种幂律关系: $L \approx c - k \cdot \log M$, 其中 $c$ 和 $k$ 是常数.这种可预测性非常有价值. 它意味着 Engram 是一个**可靠的 "性能旋钮"**. 模型开发者可以根据这条缩放曲线, 预估投入多少内存资源大概能换来多少性能提升, 从而做出更明智的成本效益决策.

另一方面OverEncoding 的曲线也显示, 增加内存容量能降低损失. 这说明了利用 N-gram 内存本身就是有益的. 而Engram 的曲线**始终位于** OverEncoding 曲线的下方, 并且**斜率更陡**.证明了 Engram 的上下文感知门控机制 (Context-aware Gating) 是其成功的关键. 简单的信息融合远不如智能的、根据上下文动态调整的融合方式高效. Engram 能从同样的内存预算中"榨取"出更多的价值.

## 4. 大规模训练

### 4.1 实验设置

作者设计了四个模型来进行"对照实验"

所有模型看到的训练数据不仅总量相同 (262B tokens), 连顺序都一样. 这排除了数据差异带来的任何影响.

模型骨干架构为 30 层, 隐藏层维度 2560. 注意力机制采用了 MLA (Multi-head Latent Attention).并且使用了mHC (Manifold-Constrained Hyper-Connections) 其中 $n=4$. 然后**优化器改为了Muon**.
1. Dense-4B作为Baseline
用于验证稀疏架构 (无论是 MoE 还是 Engram) 相对于密集架构的优越*. 由于它的激活参数量 (3.8B) 与其他稀疏模型相同, 它们的训练 FLOPs 也相同. 这个对比回答了问题: "在相同的计算成本下, 使用稀疏架构是否值得?"
2. MoE-27B作为Direct Competitor
模型配置了 72 个路由专家和 2 个共享专家 (每个 token 激活 top-k=6 个路由专家), 使其总参数扩展到 267 亿, 同时保持与 Dense-4B 相同的激活参数量.
3. Engram-27B作为实验的主角
作者将 MoE-27B 的 72 个路由专家减少到 55 个, 释放出的参数容量 (5.7B) 用于构建 Engram 表. 计算出的分配比例 $\rho$ = MoE 专家参数 / 总稀疏参数 $\approx 0.79$, 这正好落在了第 3 章发现的最优区间 ($\rho \in [0.75, 0.8]$) 内.

Engram-27B 与 MoE-27B 拥有完全相同的总参数量 (26.7B) 和完全相同的激活参数量 (3.8B). 这构成了一个 Iso-parameter & Iso-FLOPs 对比. 这个对比直接回答了核心问题: "在总参数和计算成本都一样的情况下, 将一部分稀疏预算从 MoE 专家转移给 Engram 内存, 是否能带来性能提升?"
4. Engram-40B作为一个内存增强的混合模型
它也是一个潜力探索者 (Potential Explorer). 它与 Engram-27B 拥有完全相同的计算骨架 (55 个专家) 和完全相同的激活参数量 (3.8B), 但拥有一个更大的 Engram 内存表 (18.5B vs 5.7B). 这个对比直接验证了 3.2 节的发现: "在计算成本不变的情况下, 进一步增加 Engram 的内存容量, 是否能带来持续的性能提升?" 它旨在展示 Engram 作为"可扩展内存"的潜力.

### 4.2 实验结果

实验结果如下所示:

![图片](assets/e06dd638cde3.png)

所有三个稀疏变体 (MoE-27B, Engram-27B/40B) 在所有基准测试上都显著优于同等 FLOPs 的 Dense-4B 基线.更重要的是, Engram-27B 在同等参数量和同等 FLOPs 的 MoE-27B 基线上取得了一致的提升. 有趣的是, 这些增益并不仅限于知识密集型任务 (例如, MMLU: +3.0, MMLU-Pro: +1.8, CMMLU: +4.0), 在这些任务中, 内存容量的益处是符合直觉的. 在通用推理领域 (例如, BBH: +5.0, ARC-Challenge: +3.7, DROP: +3.3), 以及代码和数学推理领域 (例如, HumanEval: +3.0, MBPP: +1.6, GSM8K: +2.2, MATH: +2.4) 观察到了甚至更显著的改进.

最后, 将模型扩展到 Engram-40B 进一步降低了预训练损失, 并在大多数基准上提升了性能. 尽管它尚未在每个任务上都严格超越 Engram-27B, 但这很可能是训练不足 (under-training) 造成的现象. 我们观察到, 在训练的末期, Engram-40B 与基线模型之间的训练损失差距仍在持续扩大, 这表明在当前的 token 预算内, 扩展的内存容量尚未完全饱和.

## 5. 长上下文训练

这一章旨在验证 Engram 架构的一个重要推论: 通过将局部依赖的处理外包给查找机制, Engram 能够释放宝贵的注意力容量, 使其更专注于全局和长距离的依赖关系, 从而在长上下文任务中表现更优.

为了获得长上下文能力, 采用了 DeepSeek-V3 中介绍的上下文扩展策略. 在预训练阶段之后, 应用 YaRN 方法进行上下文窗口扩展, 在一个 32K token 上下文长度的训练阶段中, 使用 30B tokens 的高质量长上下文数据训练了 5000 步. 比较了四种不同模型配置的上下文扩展结果.

注意,选择 Engram-27B (46k) 是因为它展现出与完全训练好的 MoE-27B (50k) 相同的预训练损失. 这创造了一个可控的 **"同等损失 (Iso-Loss)"** 设置, 确保在上下文扩展期间的任何性能差异都可归因于架构本身, 而非模型的初始质量.

使用 LongPPL 和 RULER 来评估长上下文性能, 结果如下:

![图片](assets/3699dbb3dfce.png)

**超越注意力机制的长上下文能力.** 尽管注意力机制和位置编码为上下文处理提供了结构基础, 但结果表明, 长上下文性能并不仅仅由架构先验决定. 通过观察 Engram (从 41k 步到 50k 步) 的轨迹, 即使在上下文扩展阶段控制了模型架构和计算预算完全相同的情况下, 长上下文性能也随着预训练的进展而单调提升. 这表明长上下文性能与基础模型的通用建模能力是内在耦合的. 因此, 一个严谨的架构比较必须通过**对齐基础模型的损失**, 而非仅仅对齐训练步数, 来控制这个混淆变量.

**可控设置下的架构优越性.** 在上述原则的指导下, 将 Engram 与 MoE 基线进行基准测试. 当控制了基础模型能力后, Engram 模块的效率增益变得显而易见:

Iso-FLOPs 对比: Engram-27B (50k) vs. MoE-27B (50k), 当 Engram 也被完全训练后, 它在所有指标上都进一步拉大了与 MoE 的差距, 取得了最佳性能.

Engram-27B (41k) vs. MoE-27B (50k)对比, Engram-27B (41k) 只使用了 MoE-27B (50k) 大约 82% 的预训练计算量, 并且其基础模型能力更差 (损失 1.66 vs 1.63),即便如此, 这个"未完全体"的 Engram 模型在 LongPPL 上已经能与完全体的 MoE 基线打平, 并在 RULER 任务上全面超越.

## 6. 分析

### 6.1 Engram 是否等同于增加模型深度?

这一节旨在验证一个核心假设: Engram 通过查找操作替代了模型早期层的特征组合计算, 从而等效地增加了模型的**有效深度 (effective depth)**.

#### 6.1.1 加速预测收敛

使用了LogitLens, 这个工具通过将模型中间层的隐藏状态投影到最终的 LM Head 上, 来观察模型在每一层对下一个 token 的预测是如何演变的. 通过计算中间层预测分布与最终预测分布之间的KL 散度, 可以衡量一个中间表示离"预测就绪"状态有多近. KL 散度越小, 表示越接近最终预测.

![图片](assets/8b594c0b8a80.png)

图 (a) 所示, Engram 模型的 KL 散度曲线在早期层下降得更陡峭. 这意味着 Engram 模型更早地形成了对最终预测的"确信". 通过直接查找 N-gram 信息, Engram "抄了近道", 跳过了 Transformer 早期层需要进行的、从零开始逐步构建局部模式表示的计算过程, 从而加速了整个预测的收敛.

#### 6.1.2 表示对齐与有效深度

CKA 是一种衡量两个表示空间 (例如, 不同模型或不同层的激活) 结构相似度的工具. CKA 值越高, 表示越相似. 图 (b) 和 (c) 所示, 当比较 Engram 模型和 MoE 基线模型的层级表示时, CKA 相似性热图的高亮对角线明显向上偏移.

例如, Engram-27B 的第 5 层的表示, 与 MoE 基线的第 12 层的表示最为相似."向上偏移"意味着 Engram 模型的浅层在功能上等同于 MoE 模型的深层. 这完美地验证了假设: Engram 通过外包低级任务, 使得其每一层都可以处理比 MoE 模型同等层更高级、更抽象的信息, 从而**有效地加深了模型**.

### 6.2 结构消融与层敏感性

这一节通过"拆零件"的方式来探究 Engram 架构中哪些组件是关键的, 以及模块的放置位置有何影响.在一个 3B 的 MoE 模型上进行消融实验, 以损失为评估指标.
发现 1: 注入位置至关重要
将单个 Engram 模块从第 1 层到第 12 层, 发现放置在**第 2 层**时效果最好, 性能随着注入深度的增加而下降.

早期注入能尽早分担局部模式识别任务, 避免骨干网络浪费计算, 效果最好.

晚期注入, 由于门控优势, 能利用更丰富的全局上下文信息 () 进行更精确的门控.

第 2 层是这个Trade-off的最佳单点. 而采用**分层注入** (如在第 2 层和第 6 层) 的策略则能更好地结合两者的优点, 效果更优, 同时也对系统预取更有利.
发现 2: 哪些组件最重要?
**多分支融合**: 证明了 2.4 节中为每个分支设计独立 Key 投影的策略是有效的, 它允许不同分支差异化地利用内存信息.

**上下文感知门控**: 证明了这是 Engram关键组件, 没有它, Engram 无法有效处理噪声和歧义.

**分词器压缩**: 表明在 N-gram 层面统一语义是提升效率的关键第一步.

相比之下, 移除轻量级卷积层的影响很小, 说明它只是一个辅助性的优化.

### 6.3 敏感性分析

这一节通过在推理时直接"关闭" Engram 模块来观察模型性能.

**事实知识任务 (TriviaQA, PopQA)**: 性能**灾难性崩溃**, 仅保留 29%-44% 的性能.

**阅读理解任务 (C3, RACE)**: 性能**保持良好**, 保留 81%-93% 的性能.

## 7. 相关工作

### 7.1 N-gram 建模与嵌入缩放

首先追溯了 N-gram 模型的起源 (香农), 指出其核心思想是利用局部历史进行预测.提到了 FastText, 它将 N-gram的思想成功地引入到现代的词向量表示学习中, 保留了 N-gram 查找的高效性. 然后提到了与 Engram 最相关的一系列近期工作, 它们都试图通过扩展嵌入来增加模型容量, 并且很多都借鉴了 N-gram 的思想.

**SuperBPE, SCONE**: 针对高频模式, 前者通过词表合并, 后者通过辅助编码模型.

**OverEncoding, Byte Latent Transformer (BLT)**: 采用哈希 N-gram, 前者在 token 层面, 后者在 byte 层面. 这些工作共同证明了通过 N-gram 表示来扩展参数是一种计算开销极小且有效的策略.

Engram 与这些工作的两个根本不同之处:

先前工作 (如 SCONE, OverEncoding) 要么引入了额外的训练计算成本, 要么没有在严格公平的条件下进行验证. Engram 将"条件化内存"视为建模基本组件, 并通过"稀疏性分配"框架, 在iso-parameter 和 iso-FLOPs 条件下, 证明了其优于 MoE 基线.

现有方法 (如 OverEncoding, SCONE) 通常将 N-gram 嵌入严格地放置在输入层 (Layer 0). 这种设计串行化了内存访问和计算, 无法实现优化. Engram 策略性地将内存注入到更深的层, 从而可以利用前面层的计算来overlap 内存访问的通信延迟. 此外, 提出利用 Zipf 分布和硬件内存层次结构进行优化.

### 7.2 MoE

简要回顾了 MoE 的发展历史, 从 Shazeer et al. (2017) 的开创性工作开始, 到 GShard, Switch Transformer, GLaM 等一系列推动 MoE 走向超大规模的工作.特别提到了 DeepSeek-MoE, 指出其通过细粒度专家分割和共享专家隔离等技术, 实现了优于同等激活参数密集模型的效率.

本节虽然没有直接说明, 但其意图是明确的: MoE 代表了"条件化计算"这条路线的最高水平. Engram 并非要否定或取代 MoE, 而是选择与这条路线上最强大的代表进行合作与互补.

### 7.3 Memory Network

分了两种类型:

参数化内存 (Parametric Memory): 将大规模、稀疏的键值存储直接集成到模型层中. 知识存储在模型的参数里.例如PKM, PEER, Selfmem, Memory+, Ultra-Mem.Engram 属于这一类. 它通过一个巨大的、可学习的嵌入表 (参数) 来存储知识.

非参数化内存 (Non-parametric Memory): 将知识存储与模型处理分离开. 模型在推理时从一个外部的、可编辑的知识库 (如文本数据库) 中进行检索. 这种方法通常被称为RAG. 优势是知识可以方便地更新, 无需重新训练整个模型.

通过这个分类, 作者清晰地界定了 Engram 的技术定位. 它不是 RAG, 而是对模型内部参数化记忆的一种高效扩展. 这两种方法各有优劣, RAG 灵活性高、易于更新, 而 Engram 这种参数化内存的访问速度更快、与模型集成更紧密.

### 7.4 知识存储机制

回顾了近年来关于 Transformer 内部机制的研究, 特别是"前馈网络 (FFN) 作为键值记忆"这一重要假说, FFN 的第一层充当模式检测器 ("keys"), 第二层则将特定信息投射到残差流中 ("values").基于上述发现, 诞生了 ROME, MEMIT 等模型编辑算法, 它们可以直接修改 FFN 中的权重来更新事实, 而无需重新训练.

Engram 提供了一个更直接、更高效的实现方式, 将隐式的、通过计算实现的知识存储, 变为了显式的、通过查找实现的知识存储.

## 9. 一些数学上的解释

去年国庆期间, 正好读到一些范畴论的书, 写了一篇文章[《谈谈 Hierarchical Sparse Attention (HSA)》](https://mp.weixin.qq.com/s?__biz=MzUxNzQ5MTExNw==&mid=2247496275&idx=1&sn=5f8a8d8efff22033d3f2aed8a5844e53&scene=21#wechat_redirect), 其中谈到了Nerve构造一个高阶态射来实现模型进一步稀疏化的想法. 然后DeepSeek很巧妙的直接用N-gram来构建了.

### 9.1 Nerve构造

米田引理 (Yoneda Lemma) 是范畴论的基石. 它的核心思想是: **一个对象完全由它与范畴中所有其他对象的关系 (即"箭头"或"态射") 所决定.** 换句话说, 你不需要知道一个对象的"内部构造", 只需要知道它如何与外界"交互"(即所有射向其他对象的箭头集合), 就能完全理解它.

因此我们对于一个Token可以看作由它的Attention所决定的. 于是token和attention某种意义上可以构成一个范畴. 这就进一步引出了范畴论中的Nerve[2](神经)构造.

范畴论中, 一个小范畴 $C$ 的神经(nerve) $N(C)$ 是一个由 $C$ 的对象和态射构造出的单纯集. 这个单纯集的几何实现是一个拓扑空间, 称为范畴 $C$ 的分类空间. 这些紧密相关的对象可以利用代数拓扑学(最常用的是同伦论)的方法, 为一些我们熟悉且有用的范畴提供信息.

范畴的神经常被用来构造模空间(Moduli spaces)的拓扑版本. 如果 $X$ 是 $C$ 的一个对象, 它的模空间应该以某种方式编码所有同构于 $X$ 的对象, 并记录这些对象之间的各种同构关系. 这可能会变得相当复杂, 特别是当对象有很多非恒等自同构时. 神经提供了一种组合的方式来组织这些数据. 由于单纯集有很好的同伦论, 我们可以提出关于不同同伦群 $\pi_n$ 意义的问题. 我们希望这些问题的答案能为原始范畴 $C$ 或相关范畴提供有趣的信息.

模空间(Moduli spaces)
模空间 (Moduli space) 是代数几何中的一个核心概念. 它是一个几何空间, 其上的每一点都对应着一个特定类型的代数几何对象的同构类. 模空间的研究旨在为这些几何对象的分类问题提供一个统一的框架, 并通过研究模空间本身的几何和拓扑性质, 来反过来理解这些几何对象的性质.模空间的核心思想是**“为对象建立一个几何目录”**

例如语言中的同义词, 相同含义的内容通过不同国家语言的描述等, 我们都可以看作它们为背后蕴藏的知识本身的同构类. 当对象之间存在复杂的等价关系(同构)时, 模空间本身就成了一个复杂的"空间". 神经构造(Nerve Construction)正是处理这种复杂"等价关系网"的系统性工具.

例如, 对于一个简单的三个Token之间的Attention(态射)如下:

$A$ 到 $B$ 的attention为 $f$, $B$ 到 $C$ 的attention为 $g$. 而 $A$ 到 $C$ 的attention可以看作 $f, g$ 的复合 $g \circ f$. 对于由 $N$ 个Token和它的Attention构建的一张态射图上来看, 我们能否进行某种神经构造,避免任意两个token之间的attn运算来构造?

令 $C$ 为一个小范畴. $N(C)$ 的每个**0-单纯形**对应 $C$ 中的一个对象. $N(C)$ 的每个**1-单纯形**对应 $C$ 中的一个态射 $f: A \to B$, 即任意一个token 构成一个0-单纯形, 它们之间的Dense Attn作为态射, 构成一个1-单纯形. 如上所示三个token $A, B, C$, 现在假设有可复合的态射 $f: A \to B$ 和 $g: B \to C$. 那么我们有它们的复合 $g \circ f: A \to C$. 这个图暗示了我们的做法: 为这个交换三角添加一个**2-单纯形**. $N(C)$ 的每个2-单纯形都来自于这样一对可复合的态射.

**一般情况:** $N(C)_k$ (k-单纯形的集合) 由 $C$ 中 $k$ 个可复合态射组成的序列构成:

$$A_0 \xrightarrow{f_1} A_1 \xrightarrow{f_2} \cdots \xrightarrow{f_k} A_k$$

为了完成 $N(C)$ 作为单纯集的定义, 我们还必须指定面映射(face maps)和退化映射(degeneracy maps).

**面映射** $d_i: N(C)_k \to N(C)_{k-1}$ 通过以下方式给出:

当 $0 < i < k$ 时, 通过在第 $i$ 个对象上进行态射复合.

当 $i=0$ 或 $i=k$ 时, 移除序列的首尾对象(和相应的态射).

$d_i$ 将 k-元组

$$(A_0 \xrightarrow{f_1} \cdots \xrightarrow{f_k} A_k)$$

发送到 (k-1)-元组

$$(A_0 \to \cdots \to A_{i-1} \to A_{i+1} \to \cdots \to A_k)$$

**退化映射** $s_i: N(C)_k \to N(C)_{k+1}$ 通过在对象 $A_i$ 处插入一个单位态射(identity morphism)来给出.

**函子式定义:** 单纯集也可以被看作从 $\Delta^{\text{op}}$ 到 $\text{Set}$ (集合范畴)的函子, 其中 $\Delta$ 是由全序有限集 $[n] = \{0, 1, \dots, n\}$ 和保序映射构成的范畴. 我们可以将范畴 $C$ 的神经描述为函子 $N(C): \Delta^{\text{op}} \to \text{Set}$:

这个描述使得函子性变得透明. 例如, 小范畴 $C$ 和 $D$ 之间的一个函子 $F: C \to D$ 会诱导出单纯集的一个映射 $N(F): N(C) \to N(D)$. 此外, 两个函子之间的自然变换会诱导出映射之间的同伦.

### 9.2 N-gram和Nerve构造

从代数结构上, N-gram 可以被看作是在一个极其简单的范畴上进行 Nerve 构造的"影子". 让我们来构建这个类比, 首先将句子转化为范畴:

考虑一个句子: "我 爱 北京 天安门". 我们可以将其看作一个非常简单的范畴, 记作 $C_{\text{sent}}$.

**对象 (Objects)**: 句子中的每个词的位置, 如 {0, 1, 2, 3, 4}.

**态射 (Morphisms)**: 对于任意两个位置 $i \le j$, 都存在一个唯一的态射从 $i$ 指向 $j$. 例如, 存在态射 $0 \to 1$, $1 \to 2$, 同时也存在 $0 \to 2$, $0 \to 3$ 等.

**组合 (Composition)**: 态射的组合是显然的. $0 \to 1$ 和 $1 \to 2$ 组合起来就是 $0 \to 2$.

这个范畴实际上是一个**偏序集 (Poset)**, 其顺序就是句子中词的线性顺序.

然后对此范畴进行 Nerve 构造: Nerve 构造 $N(C_{\text{sent}})$ 会生成一个单纯集合, 它的 k-单纯形是一条由 k 个可复合态射组成的链: $A_0 \to A_1 \to \cdots \to A_k$.

**0-单纯形**: 长度为 0 的链, 就是范畴中的对象. 在这里就是词的位置 {0, 1, 2, 3, 4}. 它们对应着 **unigram (1-gram)**.

**1-单纯形**: 长度为 1 的链, 如 $0 \to 1$. 这个链由对象序列 (0, 1) 唯一确定. 这个序列对应着 **bigram (2-gram)**, 即 ("我", "爱").

**2-单纯形**: 长度为 2 的链, 如 $0 \to 1 \to 2$. 这个链由对象序列 (0, 1, 2) 唯一确定. 这个序列对应着 **trigram (3-gram)**, 即 ("我", "爱", "北京").

**n-单纯形**: 长度为 n 的链, 如 $0 \to 1 \to \cdots \to n$. 这个链由对象序列 (0, 1, ..., n) 唯一确定, 对应着 **(n+1)-gram**.

一个 **N-gram** 恰好对应了 Nerve 构造出的一个 **(N-1)-单纯形** 所基于的**对象序列**. N-gram 捕获的是单纯形的"顶点", 而忽略了连接它们的"边"(态射).

主要对比如下表所示:

![图片](assets/952581f48e36.png)

预训练中构建的N-gram相当于在一个**全局语言范畴(The Universal Language Category)**中获取包含了语言中所有稳定存在的组合模式.其 Nerve $N(\mathcal{L})$ 就是所有可能的 N-gram 构成的单纯集合.Engram 检索到的静态嵌入 $e_t$ 代表了 $N(\mathcal{L})$ 中的一个特定的**单纯形** $\sigma$ (例如, 3-gram "我 爱 北京" 是一个2-单纯形). $e_t$ 是这个抽象单纯形的向量化身. 它是上下文无关的, 是一个"普遍真理"或"通用模式".

另一方面正如前面构造的范畴 $C_{\text{sent}}$, 它是一个**局部上下文范畴**. 它代表了模型在处理到 `t` 时, 已经构建起来的特定语义空间, $h_t$ 是这个范畴中的一个核心对象.

Engram 模块的目标, 就是尝试将 $N(\mathcal{L})$ 中的一个静态单纯形 $\sigma$ (由 $e_t$ 代表) **"包含"** 到当前的局部上下文 $C_{\text{sent}}$ 中. 在范畴论的语言中, 这个"包含"操作可以被想象成一个**函子 (Functor)**, 我们称之为**包含函子(Inclusion Functor)** 这个函子试图将单纯形 $\sigma$ 及其结构映射到 $C_{\text{sent}}$ 中.

但是一个普遍的模式并不总是适用于一个特定的上下文. 如何检验这个包含函子 $\iota$ 在当前上下文 $C_{\text{sent}}$ 中的"自然性" (Naturality)?

在范畴论中, **自然变换 (Natural Transformation)** 是衡量两个函子 `F` 和 `G` (它们在相同的源范畴和目标范畴之间映射) 之间"一致性"或"兼容性"的工具. 一个自然变换 $\eta$ 为源范畴中的每个对象 `X` 提供一个态射 $\eta_X: F(X) \to G(X)$, 并且这些态射必须与范畴的结构"和谐共存"

而Engram恰好在门控计算中使用了一个类似于Transformer Attention的结构. 即为了比较来自两个不同世界 ($N(\mathcal{L})$ 和 $C_{\text{sent}}$) 的结构, 我们需要将它们投影到一个共同的语义空间 `S` 中. 这个空间就是 `d` 维的"Query/Key"空间.

$W_K$: 这是一个函子 $F_K: N(\mathcal{L}) \to S$. 它将静态单纯形 $\sigma$ (的嵌入 $e_t$) 投影到对齐空间 `S` 中, 得到其"静态"表示 $k_t = W_K e_t$.

$W_Q$ (即对 $h_t$ 的处理): 这可以看作另一个函子 $F_Q: C_{\text{sent}} \to S$ 的输出. 它将动态上下文 $C_{\text{sent}}$ 的状态 $h_t$ 投影到同一个对齐空间 `S` 中.

然后通过在对齐空间 `S` 中计算了动态上下文表示 $h_t$ 和静态单纯形表示 $k_t$ 之间的几何对齐程度. 并将对齐度转化为一个 `[0,1]` 的"自然性"得分 $\alpha_t$. 只有在自然性检验通过时, 才允许单纯形 $\sigma$ 的信息(由 $v_t$ 携带)被包含进当前上下文.

当前的 Engram 主要基于 N-gram, 这对应于 Nerve 构造中线性的, 连续的单纯形 (k-simplices).语言中充满了非连续的依赖, 例如 "把...拿出来", 或者动宾短语的主谓分离. 以更高阶的视角, 我们是否可以设计一种机制来捕捉这些"带洞"的单纯形? 然后将它们哈希到一个高阶的 Engram 表中?

这将使得 Engram 的记忆更具泛化性和组合性? 而这些函子本身就是一个更高阶的Attention.

另一个演进的方向是, 当前的Engram是一个静态的预训练构成的结构, 但未来在线学习等能力的需求下, 很多知识和语言模式是动态的或领域特定的. 例如在一次特定的对话或一篇特定的文档或特定的Agent工作流中, 会形成临时的, 局部的语言范畴. 例如某种编程语言下的一个代码仓库? 其中的一些函数就成为一个重要的单纯形. 这些如何来构建一个高阶的Engram表? 特别来说它对于Agent的执行成功率等会有很大的提高.

当然, 未来如何和一些领域相关的RL结合起来也是一个值得探讨的话题. 不同的领域有自己独特的语言范畴和高频单纯形(术语, 代码模式等), 这些领域的Engram模块如何构建, 如何做到一个基础模型, 多个领域即插即用来构建?

最后是一些可学习的Nerve Key, 让模型**学习**生成查询 Engram 的"键". 这个键本身就是一个向量, 捕捉了模式的**语义**, 甚至是一些非语义的内在逻辑结构, 而非其词法形式. 在一个更稀疏的高维范畴上进行一系列Attention运算, 然后再降维Decoding, 这样的一个好处是可以摆脱自回归模型Token-by-Token的处理, 例如基于Nerve视角的一个Diffusion LLM? 可以称为Nerve-Diffusion Language Model (ND-LLM).

ND-LLM 的核心思想是: **将扩散过程分解为两个相互关联但又功能分离的部分**:

结构 Nerve 的扩散 (Diffusion of the Structural Nerve): 对语言的**组合骨架**进行扩散. 这不是对 token 本身, 而是对"token 之间如何连接"的抽象关系构成的一系列单纯形进行加噪和去噪.

内容填充的扩散 (Diffusion of the Content Filling): 在给定的结构骨架下, 对填充到这个骨架中的具体语义内容进行扩散.

ND-LLM 的生成过程是一个**两阶段的去噪(denoising)过程**:

第一阶段: 结构去噪 (Structural Denoising)

第二阶段: 内容去噪 (Content Denoising)

即模型的输入从Token变为一个噪声化的单纯形嵌入序列 $S_T$, 它的Embedding为 $E_{S_T}$. 注意力机制学会的是**单纯形之间的组合规则**, 然后经过 `T` 步去噪后, 我们得到一个清晰的结构嵌入序列 $S_0$. 这是一个"语义骨架", 它规定了句子的流畅性和语法结构, 但还没有填充具体内容. 例如, 它可能代表了 "一个<名词>正在<动词>一个<形容词>的<名词>" 这样的抽象模板.

在第一阶段给定的清晰结构骨架 $S_0$ 的**强力约束**下, 从噪声化的内容 $C_T$ 中去噪, 预测出最终的 token 序列.

### 10.1 关于大EP的问题

其实最近一年来, 很多做ScaleUP的同学都在阐述一个愿景, 未来会有数千个专家的模型,但是训练本身很容易造成大量的Dead Expert, 简单的扩展MoE的数量达到参数规模的扩展并不太可行. Engram使得模型有了另一个可以scale的维度. 因此也不需要复杂的超大规模的ScaleUP网络了.

当前Engram可以使用CPU DRAM进行Offload, 甚至根据Zipf定律, 可以进一步cache到NVMe SSD盘上.  但是另一方面我们注意到KVCache这些业务本身也会大量占用这些存储空间, 未来是否会像传统的搜推广模型那样, 出现独立的通用CPU计算节点来Offload?

另外有一个很有趣的视角, 现在的MaaS卖Token的逻辑, 实质上和CDN卖带宽的逻辑类似的. 用户换个API就可以轻松的迁移到其它平台上, 因此这种商业模式注定会在激烈的竞争中只有相对微薄的利润, 虽然有巨大的营收, 但也同时存在巨大的基础设施投资成本和流动性风险.

把一些用户行为像搜广推那样做成一个Per-User的Engram表, 增加用户粘性. 特别是在ToC市场上, 这种处理是很有必要的. 现在很多开源模型在大多数ToC任务上, 终端用户一般分辨不出太大的差异, 大致就像前几天几个大佬开会讲的那样92分和98分有多少普通人能够分辨出来? 更多的是一些情感上, 语气表达上等一系列的千人千面的需求. 基于Engram这个视角继续做下去, 特别是对一些ToC的App看上去还是挺有吸引力的. 这些都会成倍的增加Embedding Table的容量.
