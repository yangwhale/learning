### 1. 偏序结构：从递归到并行

NLP任务很多，例如机器翻译(MT)/问答(QA)/命名实体识别(NER)/情感分析(SA)等，通常这些任务的输入语料和输出结果都是`变长`的。例如我们输入`I like eRDMA`是三个词，而输出是`我 喜欢 弹性 RDMA`输出是四个词。

对于输入输出都是变长的序列，我们能不能够用一个定长的状态机作为中间件来使用? 也就是说我们构建一个编码器(Encoder),把变长的输入编码成一个定长的状态张量，然后再构建一个解码器(Decoder)根据状态张量产生变长序列的输出，这就构成了一个简单的Encoder-Decoder结构。

![图片](assets/ab1bac276a98.png)

另一方面语言本身的词序和语法构建了一个偏序结构，但是通常又有定语后置，补语和各种从句等方式导致整个语序并不完全满足偏序结构，因此就需要注意力机制来进行处理，我们先开始从Seq2Seq介绍，然后逐渐切换到注意力机制，最后再导出Transformer模型架构。

#### 1.3 Transformer

可以看到整个NLP的发展，很多刚入行的算法工程师对Transformer解决的关键问题并不清楚，过分的关注于Attention的实现，解释Q/K/V是什么。这也是本文前面这几段补充的内容。
事实上Transformer解决的关键问题是它既能够对变长序列处理，并且捕获大跨度的序列信息，又能够并行计算使得超大规模模型训练成为可能。

1.3.1 输入输出数据的表示
表示翻译自Representation，英文更能体现它的意义，这一节主要包含`Word Embedding`和`Positional Encoding`

`Word Embedding`是一个向量化的编码，其数学本质是构建一个单射且保证同构。实际上就是我们在前两章讲的`可表函子(Representable Functor)`

即选择C中的每一个对象,我们可以获得一个从C到Set的函子, 这种指向Set的保持结构的态射，通常被成为一个表示(representable).形式化的定义如下：

一个共变(或反变)函子, C is locally small category, 如果存在一个对象,使得和自然同构(如果F为反变函子，则为),则称函子F是可以被对象A表示的.

例如：`东京 = 北京 - 中国 + 日本`把它映射到一个高维向量空间然后通过向量运算得到类似的结果。

`Position Encoding`：在NLP任务中词所在的位置和顺序非常重要，RNN天然的和词序配合在一起，而在Transformer中为了并行性，RNN被多头注意力(Multihead-Attention,MHA)机制替代， 如何在并行计算的框架中维持原有数据的顺序？ 常见的办法就是对位置信息进行编码并加载到数据上。位置编码函数需要保证如下几个性质：

在每一次迭代中，词的位置编码具有唯一性

任何两次迭代间，词的相对位置距离不变

编码具有确定性

编码结果通常独立于语句的长度

Transformer论文中的编码函数如下所示：

```
import numpy as npimport matplotlib.pyplot as plt def getPositionEncoding(seq_len, d, n=10000):    P = np.zeros((seq_len, d))    for k in range(seq_len):        for i in np.arange(int(d/2)):            denominator = np.power(n, 2*i/d)            P[k, 2*i] = np.sin(k/denominator)            P[k, 2*i+1] = np.cos(k/denominator)    return P P = getPositionEncoding(seq_len=2048, d=12288, n=100)print(P)xx = plt.matshow(P)plt.gcf().colorbar(xx)
```

1.3.5 残差和LayerNorm
残差计算和ResNet作用一致

而LayerNorm的作用是降低“Covariance shift”，改善训练的稳定性避免梯度消失或者梯度爆炸，有助于训练更深的网络。
本质就是对该层数据求平均和标准差，然后进行归一化的操作。

![图片](assets/eb8a406b4153.png)

```
class LayerNorm(nn.Module):    def __init__(self, d_model, eps=1e-12):        super(LayerNorm, self).__init__()        self.gamma = nn.Parameter(torch.ones(d_model))        self.beta = nn.Parameter(torch.zeros(d_model))        self.eps = eps    def forward(self, x):        mean = x.mean(-1, keepdim=True)        std = x.std(-1, keepdim=True)        out = (x - mean) / (std + self.eps)        out = self.gamma * out + self.beta        return out
```

论文中采用后置LayerNorm(Post-LN)，作用类似于标准的BatchNorm即通过神经网络后，数据的分布会发生改变，通过后置LN能够控制好输出数据的分布，但是注意这个地方为什么没有在batch维度做Normalization而是要在Layer维度，留个思考题？

LayerNorm的位置也有讲究，前置LayerNorm(Pre-LN)，即在每个子层之前(例如Attention/FFN之前)应用LayerNorm，目的是改善训练的稳定性避免梯度消失或者梯度爆炸，有助于训练更深的网络。

后续针对LayerNorm也有一些优化，例如RMSNorm等..

### 2.Transformer带来的各种墙

第一节我们简述了从Seq2Seq到Attention再到Transformer，各个组件演进的过程及在模型中扮演的角色。

我们注意到在Transformer模型中有大量的访存密集型算子，例如LayerNorm/ReLU/Softmax/Transpose等，这些对内存带宽需求极大。大模型的参数/Activation/Optimizer又对显存容量有很大的需求，虽然内存容量有TP拆分或者Deepspeed的一系列Offload方案在解决，本质也是靠通信和算力来置换，而另一方面全连接层也有大矩阵乘法等计算密集型算子。

多卡分布式并行涉及到TP并行时的集合通信同步阻塞需要超大带宽和超低延迟才能保证MFU,通常TP被约束在单机八卡之内的NVLINK上。而DP又受到BatchSize的影响，以及超大规模下DP导致每卡的集合通信量较小等影响无法大规模的Scale-out。而PP并行又涉及到流水线空泡以及每个GPU在SPMD模式下又要异步执行的问题，这部分Google Pathways对于PP调度做了一系列尝试构建MPMD是一条出路。当然通过MoE置换FF层的算力复杂性也是一条路。

#### 2.4.1 Welford's在线算法

我在2018年为思科构建一个大数据分布式流处理引擎的时候就使用过一个基于Welford's online algorithm的优化，并且在很多需要统计数据的地方使用，基于Golang的版本可以参考：

https://github.com/zartbot/ztrace/blob/main/stats/describe/stats.go

具体算法如下：

```
# 通过一个更新函数迭代计算计数器/Delta/M2来更新均值方差统计def update(existing_aggregate, new_value):    (count, mean, M2) = existing_aggregate    count += 1    delta = new_value - mean    mean += delta / count    delta2 = new_value - mean    M2 += delta * delta2    return (count, mean, M2)# Retrieve the mean, variance and sample variance from an aggregatedef finalize(existing_aggregate):    (count, mean, M2) = existing_aggregate    if count < 2:        return float("nan")    else:        (mean, variance, sample_variance) = (mean, M2 / count, M2 / (count - 1))        return (mean, variance, sample_variance)
```

关于LayerNorm在GPU上基于Welford's的优化可以参考OneFlow的一篇文章： https://zhuanlan.zhihu.com/p/443026261

#### 2.4.2 Pre-LN vs Post-LN

关于LayerNorm放置在哪个位置也是值得探索的一个问题，即LN放置在Attention和FFN之前我们称作Pre-LN，例如Llama就采用了这样的方式，而标准的Transformer架构采用了Post-FN

![图片](assets/8276c241d556.png)

他们各自的优势：

Pre-LN,在《On Layer Normalization in the Transformer Architecture》的观点是：模型的参数是随机初始化时需要对学习率Warmup的过程，Post-LN如果没有warmup会容易导致梯度爆炸

![图片](assets/2f5c7fbcc4ae.png)

Pre-LN 通过标准化的输入，有助于解决梯度消失或爆炸的问题，同时看上去是可以让模型训练更深的网络,同时Pre-LN收敛速度也比Post-LN快很多

![图片](assets/3a5ec328b71b.png)

Post-LN, 也是标准的实现方式，类似于传统神经网络中的Batch-Normalization，有助于控制每层的输出分布。 注意在大模型中为什么不能使用Batch-Norm，而是使用Layer-Norm，这两个维度的选择上最根本的原因是在NLP任务中Batch长度没对齐，所以BatchNorm就不能用了。

看上去Pre-LN能省算力能快速收敛，Llama也在用，国内基本上也开始抄作业，但是从数学的角度上再来看看这个问题，对于任何一个函数来表示Attention或者FFN的计算，我们来看整个Pre-LN的迭代过程

实际上是得到了一个更宽更浅的模型，这样配合一定的稀疏性，例如LoRA微调这些，模型的质量反而会下降，总体来看为了一个算力上的优势把模型的效果降低，这样的取舍是否得当？我们需要辩证的来看，而不是抱着抄作业的心态。

后续Token输出时间(Time Per Output Token,TPOT)：模型生成后续Token的时间，通常的需求是要满足人类阅读速度的需求，即每个Token 100ms

总延迟：

吞吐量：推理服务如何并行的为多个用户产生Token的最大数量

通常在大模型推理过程中的优化策略如下：

算子融合，例如FlashAttention/PageAttention

量化：降低精度，例如采用INT8/INT4等

压缩：利用稀疏性/知识蒸馏/LoRA/QLoRA等

并行性：利用张量并行或者流水线并行降低推理延迟

#### 3.2 KV Cache

由于模型的生成是一个自回归过程，因此推理阶段常用的一个策略就是使用KV-Cache。
首先针对输入的一个Prompt序列，为每个Transformer层都生成Key和Value的Cache。

![图片](assets/19d8b404092e.png)

后续Token推理过程中把前面Token的Key和Value直接从Cache中读取降低重复计算的复杂度。

Speculative Decoding 推测解码， 利用一个小的速度较快的模型先解码多个Token， 并将它们作为一个Batch输入到一个大模型中。 如果小模型的预测结果准确，大模型也认可，那么久能够在单个Batch内完成多个Token解码，反之丢弃batch，大模型进行解码。

#### 3.3 PageAttention

KV Cache也会带来一个问题，显存中碎片太多以及不同的任务如何管理，拿么一个简单的做法就是模仿操作系统那样去分页管理

### 5. 一些新的想法

#### 5.1 Adaptive BatchSize & Optimizer

从训练的角度来看，如何充分利用优化器和观测梯度进行自适应的BatchSize，这个视角是从如何让模型更快的收敛和模型稳定性的角度来看待的，而我们现在训练过程中更多的是在考量单次迭代的最大MFU采用固定BatchSize，或者从另一个维度来看就是Adaptive Gradient Accumulation的问题。

对于训练中的梯度和优化器的分析是我们后面需要关注的一个问题，特别是在从Scaling Law上数据无法满足大模型规模的时候，这是我们值得探索的一个方向。

#### 5.2 模型的深度和宽度

模型的宽度也是的复杂度，另一方面从梯度更新的角度来看Pre-LN本质上也是在加大模型的宽度。同时我们发现大模型的参数实际上并没有被充分利用，还有很多潜力可以挖，如何通过不增加算力增加模型宽度，同时维持中等的模型深度或许是一个更好的选择。例如使用MoE但是把Top-K Gating选择更多的Experts是不是一条路？

#### 5.3 GNN+LLM

联结主义、符号主义的融合来看，基础模型本来就是一个预层范畴，推理过程可以看作是一个DFS搜索，拿么是否可以将这个FM范畴的态射变成一个图，然后在进一步在图上做推理？或者是基于图结构和范畴论中的态射可组合性以及自然变换等构建态射损失函数来生成数据进行训练？

![图片](assets/0e8774cad695.png)

另外总是觉得Attention和Ray Tracing有一些相似的地方，我们是否可以把Attention的向量空间投射到另一个空间，然后通过BVH来降低运算复杂度呢？只是简单臆想一下。

#### 5.4 硬件结构设计

在国内算力被限的时候，本文前述章节详细分析了大模型的各个组件的算力需求和瓶颈，以及学术界和工业界的一些工作，这些工作或者未来的一些新的算法如何能够通过算力约束的硬件更好的支撑？这是我们需要考虑的问题，没有GraceHopper如何通过一些大量的低算力组件构建Scale-Up和Scale-Out的能力。PIM/PNM这样的器件配合算法上的调整是否可以走出一条不依赖GPU的路？

通盘从数学/算法/工程/硬件上全方位去看待这个问题，这才是真正的我们需要考虑的软硬件结合的路，而不是简单的通过硬件优化模型算法。
