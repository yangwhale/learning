## 1. 什么是Attention Sink

ASink简单的说模型倾向于将大量概率权重分配给特定的token (通常是初始token), 尤其是在没有其他token在语义上相关时. 我们先从现象入手, 再逐渐展开到一些经验上的缓解办法. 并在稍后的章节中分析具体成因.

### 1.1 从现象谈起

#### 1.1.1 首Token形成ASink

最早关注到 ASink 的论文来自于StreamingLLM 《Efficient Streaming Language Models with Attention Sinks》[1]. 当然这篇论文并不是完全的在介绍ASink, 而是尝试通过 SWA 解决 Long Context 问题时观察到的现象. 下图是在Llama2-7B模型中针对256句子的前16个token的平均注意力Logits的可视化:

![图片](assets/ca3537cb7f91.png)

图中的每一个小方格都是一张热力图

**横轴(X轴)**: 代表句子中的Keys, 从左到右是第0个词到第15个词.

**纵轴(Y轴)**: 代表当前正在处理Query, 从上到下是第0个词到第15个词.

**颜色**: 颜色越亮(偏红), 代表注意力分数越高; 颜色越暗(偏蓝), 代表分数越低.

在最开始的两层(Layer0 / Layer1), 注意力主要集中在对角线附近. 这意味着, 当模型处理一个词时, 它最关心的是离它最近的前面几个词. 这是一种很符合直觉的“局部”注意力模式, 就像我们理解一句话时, 当前词的含义和它紧邻的词关系最大. 而在Layer1已经有向首token聚集的趋势了.

从第2层开始, 往上一直到第31层, 画面变了. 除了对角线的局部注意力外, 每一张图的第一列都非常亮. 这意味着, 无论模型当前在处理句子中的哪个词(不管是第5个, 还是第10个, 甚至是最后一个词), 它都会分出相当大的一部分注意力去“看”句子的第一个词(token 0).

#### 1.1.2 中间Token形成ASink

另一方面, 在论文 《Unveiling and Harnessing Hidden Attention Sinks: Enhancing Large Language Models without Training through Attention Calibration》[2] 中揭示中间的某些 token 也有可能成为 Sink Token:

### 1.2 ASink的影响

#### 1.2.1 Sink token对注意力分布的影响

通常为了支持更长的context会引入基于滑动窗口的注意力(SWA). 通常采用一个预先固定的窗口大小进行滑动, 当context 超过窗口大小时, 就会丢弃前面的token 通过移动窗口来接受新的token. 在StreamingLLM中构建 SWA 时观察到了如下的情况, 该语言模型在长度为 L 的文本上进行预训练, 预测第 T 个 token(T >> L).

![图片](assets/d31da69e4576.png)

(a) 密集注意力 (Dense Attention): 时间复杂度为 , 缓存大小不断增加. 当文本长度超过预训练长度时, 其性能下降.

(b) 窗口注意力 (Window Attention): 缓存最近L个token的KV. 虽然推理效率高, 但一旦初始token的键和值被驱逐, 性能会急剧下降.

(c) 带重计算的滑动窗口 (Sliding Window w/ Re-computation):为每个新token重新构建最近L个token的KV状态. 虽然它在长文本上表现良好, 但其 的复杂度使其相当缓慢.

(d) StreamingLLM : 保留 ASink (几个初始token)以稳定注意力计算, 并结合最近的token. 它既高效, 又能在扩展文本上提供稳定的性能.

这是一个非常有趣的现象, 即保持初始 token 的 KV 状态可以显著恢复窗口注意力的性能. 并且StreamingLLM作者做了另一个实验:测试不同模型在 20K token 文本上的 PPL.

![图片](assets/1c21933ba668.png)

蓝色线 (Dense Attention): 刚开始表现不错. 但当文章长度超过模型的“记忆极限”(比如Llama-2的4096个词), 困惑度变得极高.

橙色线 (Window Attention): 刚开始也很好. 但当文章长度超过了它设定的“短期记忆”窗口大小(比如缓存了2048个词)时,  模型瞬间“失忆”, 开始胡言乱语.

红色线 (StreamingLLM): 从头到尾都非常平稳, 困惑度一直保持在很低的水平. 这说明即使文章很长, 它也能一直保持稳定.

对于初始 token 的影响, 有两种可能的解释:

它们和语义相关

它们和位置相关, 模型学习了对这些 token 绝对位置的信息

## 2. ASink的成因

对于ASink 成因, 过去有多篇论文从多个角度分析. 在论文《When Attention Sink Emerges in Language Models - An Empirical View》[5]进行了详细的分析, 主要观点如下:

**Sink 是"炼"出来的, 不是"生"出来的**: 论文中描述"Attention sink emerges after LMs are trained effectively on sufficient training data." ( AttnSink 是在模型被有效训练后涌现的). 这说明它不是一个静态的、设计上的缺陷, 而是模型在学习过程中为了优化目标而发展出的一种动态特性.

**Sink 的可塑性:**  "The sink position is highly related to the loss function and data distribution and can be shifted..." ( Sink 的位置与损失和数据高度相关, 并且可以被移动). 如果 Sink 是模型架构的固有属性, 那么它应该总是出现在第一个位置. 但实验证明, 通过操纵训练数据(比如总是在第三个位置放一个特殊符号)或改变损失函数(比如让前三个 token都不参与预测), 就可以把 Sink "引诱"到新的位置. 这暗示了 Sink 是一个功能性的角色, 而非结构性的.

**Sink 的真实身份: 键偏置(Key Biases)**, "Attention sink acts more like key biases... not contribute to the value computation." ( AttnSink 更像键偏置, 不对值的计算做贡献).

Sink token 的 Key 向量在几何上与其他所有 token 的 Query 向量都有很小的角度, 因此能自然地吸引大量注意力分数.

它的值(Value)向量范数又很小, 确保了即使它吸引了大量注意力, 也不会对最终的语义计算产生大的干扰. 它就像一个"注意力黑洞", 或者说是一个专门用来存放多余注意力的"蓄水池".

**根本原因: Softmax的"锅"**: "...stems from tokens’ inner dependence on attention scores due to the softmax normalization." (源于Softmax归一化导致的内在依赖性). Softmax函数强制要求所有位置的注意力权重加起来等于1. 这就意味着, 即使模型认为当前没有任何一个历史 token是重要的, 它也必须"硬凑"出总和为1的权重. 为了解决这个约束, 模型便学会了把这些"无处安放"的多余权重全部丢给第一个 token 这个方便的"垃圾桶".

```
def sdpa(Q, K, V, S, sm_scale, sliding_window=0):    """    缩放点积注意力（Scaled Dot-Product Attention）with Sink机制    参数说明：    - Q: [n_tokens, n_heads, q_mult, d_head] - Query张量    - K: [n_tokens, n_heads, d_head] - Key张量    - V: [n_tokens, n_heads, d_head] - Value张量    - S: [n_heads] - Sink参数, 每个头一个标量值    - sm_scale: softmax缩放因子, 通常是1/sqrt(d_head)    - sliding_window: 滑动窗口大小, 0表示不使用滑动窗口    返回：    - [n_tokens, n_heads * q_mult * d_head] - 注意力输出    """    n_tokens, n_heads, q_mult, d_head = Q.shape    assert K.shape == (n_tokens, n_heads, d_head)    assert V.shape == (n_tokens, n_heads, d_head)    # 扩展K和V以匹配Q的q_mult维度（用于GQA）    K = K[:, :, None, :].expand(-1, -1, q_mult, -1)    V = V[:, :, None, :].expand(-1, -1, q_mult, -1)    # 准备Sink    S = S.reshape(n_heads, q_mult, 1, 1).expand(-1, -1, n_tokens, -1)    # 构建mask：上三角为-inf（不能看到未来）    mask = torch.triu(Q.new_full((n_tokens, n_tokens), -float("inf")), diagonal=1)    # 添加滑动窗口mask：下三角也设为-inf    if sliding_window > 0:        mask += torch.tril(            mask.new_full((n_tokens, n_tokens), -float("inf")), diagonal=-sliding_window        )    # 计算注意力分数: Q @ K^T    QK = torch.einsum("qhmd,khmd->hmqk", Q, K)    QK *= sm_scale  # 缩放以稳定梯度    QK += mask[None, None, :, :]  # 应用mask    # *** 关键步骤：将Sink拼接到注意力分数的末尾 ***    # QK: [n_heads, q_mult, n_tokens, n_tokens]    # S:  [n_heads, q_mult, n_tokens, 1]    # 结果: [n_heads, q_mult, n_tokens, n_tokens+1]    # 最后一列是Sink, 提供一个额外的"注意力目标"    QK = torch.cat([QK, S], dim=-1)    # Softmax计算注意力权重    # 注意：这里的softmax会在n_tokens+1个位置上归一化    # 分配给Sink的权重会被"浪费"掉（不贡献到输出）    W = torch.softmax(QK, dim=-1)    # *** 移除Sink列 ***    # 只保留前n_tokens列, 丢弃最后一列（Sink的权重）    W = W[..., :-1]    # 计算 W @ V    attn = torch.einsum("hmqk,khmd->qhmd", W, V)    return attn.reshape(n_tokens, -1)
```

## 4. 从数学上解释 ASink

### 4.2 ASink作为几何锚点

在论文《What are you sinking? A geometric approach on attention sink》[14]中认为:

ASink 代表了Transformer的参考系, 即其表征流形(representational manifolds)内部的坐标系. Transformer的运算依赖于 query 和 key之间的点积, 这使得角度关系对于信息流动至关重要. 参考系提供了固定的几何锚点, 允许 token 通过一致的角度关系来建立相对位置, 从而解决了在高维空间中维持稳定几何关系的挑战. 如果没有这些锚点, token 表征将缺乏一致的方向性, 使得对位置和语义关系进行可靠编码成为不可能.

自注意力机制的核心是计算 query 向量  和 key 向量  的点积: . 这个得分直接决定了 token   对 token   的注意力权重. 这表明, Transformer内部的信息流动高度依赖于向量间的**角度关系**()和**模长**().

如果没有一个固定的参考, 整个向量空间可以进行任意的刚性旋转, 而所有向量之间的点积(即内部几何关系)保持不变. 这听起来不错, 但问题在于,  token 的表示  是由其输入  和位置编码  共同决定的. 如果整个坐标系是“浮动”的, 模型很难学习到一个稳定的函数, 将一个绝对的输入  映射到一个相对的, 依赖于其他所有 token 当前“浮动”位置的输出.

建立一个参考系, 就相当于在整个旋转对称的空间中选择一个特定的“首选”坐标系. 比如, 将参考向量  对齐到某个坐标轴, 那么其他所有向量就可以用它们与这个轴的夹角来表示, 从而获得一个稳定的表示.

#### 4.2.1 参考系

在论文《Relative Representations Enable Zero-Shot Latent Space Communication》[15]表明, 在高维表征空间中, 潜在向量需要一种一致的方式来相互关联, 这通过建立**参考系**来实现, 这些参考系作为**规范坐标系**来锚定表征流形. 如果没有稳定的参考点, 模型将难以一致地确定 token 之间的关系, 距离和方向也会变得模糊不清.

从本质上讲, Transformer中的参考系是一个结构 , 其中  是表征流形 (一个光滑, 局部欧几里得的拓扑空间),  是一组作为流形上特异位置的参考点, 而  是一个将任意点与参考点相关联的映射函数, 为流形提供了一个坐标图. 这种结构提供了一种一致地测量 token 表征之间关系的方法, 而不论其语义内容或位置如何.

#### 4.3.1 几何与拓扑分析视角

通过持久同调(persistent homology)和Betti数来量化注意力网络的拓扑结构(参考附录A.1), 揭示连接模式如何跨网络深度演化. 方法借鉴了拓扑数据分析(TDA)在神经网络中应用(参考附录A.2)和TopoAE(参考附录A.3). 并使用持久同调方法来分析高维数据结构

渣注
在进行持久同调计算时, 需要一个距离矩阵, 通常定义为

如果注意力权重  很高 (接近1), 那么 token   和  在"注意力空间"中的距离  就很近.

如果注意力权重  很低 (接近0), 那么它们之间的距离  就很远. 这个距离矩阵  随后被用作构建Rips复形和计算持久同调的输入.

通过利用Ripser算法在这些距离矩阵上计算持久同调, 追踪Betti-0 (连通分支) 和 Betti-1 (环/圈) 跨层的变化. 持久性值(persistence values)衡量拓扑特征的显著性, 值越高表明结构越稳定. 这种方法为每种参考系类型揭示了特征性的拓扑签名.

**单独分析每个头:** 将每个  视为一个独立的图. 这可以用来分析是否有"特化的头".

**平均/聚合分析:** 将同一层的所有头的注意力矩阵进行平均,  (其中  是头的数量), 然后基于这个平均后的矩阵来构建图. 论文中的宏观分析(如Betti数, Fiedler值)通常采用这种方式, 以获得该层总体的结构特征.

**代数连通性 (Fiedler值):** Laplacian矩阵的第二小特征值. 值越大, 图的连接越紧密, 越难被分割.

**星形度:** 一个衡量图有多像一个理想星形图的指标.

**基尼系数:** 以量化注意力分布的不平等性.

分析结果:

#### 4.4.1 拓扑分析

![图片](assets/38091f3c41ca.png)
A. 连通分支 (Betti-0)
**含义:** 衡量注意力图在某个阈值下被分成了多少个互不连通的"小团体". 这里的数值是跨多个样本和阈值的平均值.

**LLaMA (集中式参考系):** Betti-0从头到尾都是 26.43 (变化=0).

这是一个极其稳定的结构. 注意力图的连通性在整个网络深度中保持不变. 这完美符合"集中式"的星形拓扑假设: 整个图就是一个巨大的连通体, 加上一些可能存在的孤立点, 这个整体结构从不改变.

**Qwen (分布式参考系):** Betti-0从 26.16 减少到 17.97 (变化=-8.19).

这表明在Qwen的深层网络中, 早期层中那些独立的"小团体"开始合并了. 这与"分布式"框架的假设相符: 早期模型可能建立多个局部的参考中心, 形成多个社群; 随着信息处理的深入, 这些局部信息需要被整合, 于是社群之间建立了连接, 导致连通分支数量减少.

**XLM-RoBERTa (双向式参考系):** Betti-0从 22.68 急剧减少到 1.69 (变化=-20.99).

这是一个剧烈的拓扑简化过程. 早期层中大量分散的组件, 到了网络末端几乎全部被整合进一个单一的, 巨大的连通体中. 这说明双向式框架在后期进行了大规模的信息融合, 将所有局部上下文全部汇集在一起.
B. 环/圈 (Betti-1)
**含义:** 衡量注意力图中存在多少个非平凡的"环路"或"洞".

**LLaMA & Qwen:** Betti-1始终为 0.

这两种解码器模型的注意力图在拓扑上都是无环的, 属于"树状"或"森林状"结构. 集中式的星形图是树的一种特例. 这表明信息流动是定向的, 不存在复杂的循环依赖.

**XLM-RoBERTa:** Betti-1从 19.69 急剧减少到 3.11 (变化=-16.58).

这是双向式框架最独特的拓扑特征. 早期层中存在大量的环路, 这源于其双向注意力机制, 任何 token 都可以关注任何其他 token , 从而轻易形成  这样的循环关注路径. 这种高复杂性可能允许模型在初期捕捉丰富的局部依赖. 随着网络加深, Betti-1的急剧减少表明模型正在"剪枝"和"简化"这些复杂的依赖关系, 提炼出最重要的信息流路径.
C. 拓扑持久性 (Topological Persistence)
**含义:** 衡量拓扑特征(主要是Betti-0, 即连通分支)的**"显著性"或"寿命". 值越高, 代表结构越稳定, 越不容易因尺度(注意力阈值)变化而改变.

**LLaMA:** 持久性从 0.0573 显著增加到 0.1620 (变化=+0.1046).

集中式的星形结构随着网络加深变得越来越稳固, 越来越不容置疑. 模型在深层更加依赖这个单一的参考框架. 这个特征的重要性在不断被强化.

**Qwen:** 持久性从最高的 0.2381 减少到 0.1543 (变化=-0.0838).

早期层建立的局部参考中心非常清晰且彼此分离(因此持久性最高). 到了后期, 随着这些中心开始整合(Betti-0减少), 它们之间的界限变得模糊, 整个结构的"可分离性"下降, 导致持久性降低.

**XLM-RoBERTa:** 持久性从 0.0521 减少到 0.0156.

持久性一直很低, 并且持续下降. 这说明其拓扑结构不仅复杂, 而且不稳定. 在深层, 当所有东西都融合成一个大整体后, 几乎不存在任何持久的, 可分离的子结构.
D. 注意力头特征 (Attention Head Specialization)
**含义:** 分析哪些特定的头和token在扮演"ASink"的角色.

**LLaMA:** 100% 的特化头(共120个)都集中关注文本开始标记 (BoT).

证明了其参考框架是绝对集中的. 几乎所有负责结构功能的头都指向同一个目标. 特化主要发生在第0层, 再次印证了坐标系在网络最开始就建立的假设.

**Qwen:** 只有7个头表现出特化, 并且它们主要关注的是一个逗号 (",") (占65.4%).

很好的"分布式"证据. 参考点的角色不是由特殊的BOS token 扮演, 而是由一个见的句法 token 扮演. 特化头的数量很少, 意味着这是一个更分散, 更"民主"的系统.

**XLM-RoBERTa:** 77个 头特化于开始/结束标记 (`<s>`/`</s>`).

清晰地展示了"双向式"的特点. 参考点是序列的两个端点. 有趣的是, 顶层特化发生在第18层, 而非第0层. 这暗示双向框架的建立是一个更晚近, 更深层的过程, 与Fisher信息在中间层达到峰值的观察一致.
E. 注意力标准差 (Attention Standard Deviation)
**含义:** 衡量注意力权重分布的离散程度. 标准差越大, 意味着权重分布越不均匀, 注意力越"尖锐"或"集中".

**LLaMA:** 在中间层(L21)达到最高值(0.1620).

**Qwen:** 在中间层(L14) 达到最高值(0.1316).

**XLM-RoBERTa:** 在中间层(L12)** 达到最高值(0.0950).

**解读:**

所有模型的注意力集中度都在**中间层达到峰值**. 这可能是一个普遍现象, 中间层是模型进行最密集信息处理和提炼的区域.

LLaMA的集中度最高(0.1620), 这符合其高度依赖单一参考点的特性.

XLM-RoBERTa的集中度最低(0.0950), 这符合其早期层注意力分散(高Betti-1), 后期层进行整合的模式. 它的注意力分布整体上更平滑.

#### 4.4.2 图谱分析

![图片](assets/147a6b2e7016.png)
A . 代数连通性 (Fiedler值)
**含义:** 图的拉普拉斯矩阵的第二小特征值. 它衡量了图的连接紧密程度. 值越大, 图越难被切分成两个部分, 信息在图中的流动性越好.

**高阈值有效性:** 这个指标衡量在较高的注意力阈值下, 有多少层的注意力图仍然是完全连通的.

**LLaMA (0.04):** 只有极少数层(1/28)在高阈值下保持连通. 这说明其连接是"脆弱"的, 依赖于少数强连接(即指向BOS的连接).

**Qwen (0.25) & RoBERTa (0.38):** 有相当一部分层在高阈值下仍保持连通, 说明它们的连接结构更"健壮", 不依赖于单一的连接模式.

**早/中/晚层 & 最大值:**

**LLaMA & Qwen:** Fiedler值相对较低, 并在最开始的层(L0, L1) 达到峰值, 随后逐渐下降. 这表明信息流动效率最高点在网络初期.

**RoBERTa (55.3):** Fiedler值极其高, 远超其他两者, 且峰值出现在第1层. 这说明双向注意力机制创造了一个信息交换效率极高的网络. 任何节点到任何其他节点都可能存在短路径, 这与表1中高Betti-1(多环路)的发现相辅相成.
B. 星形度 (Star-likeness)
**含义:** 一个衡量图的结构与一个理想星形图有多接近的指标. 值越接近1, 图越像一个星形.

**低阈值 (t=0.001):** 在这个阈值下, 只有非常强的注意力连接被考虑.

**LLaMA & Qwen (约0.54):** 表现出中等的星形度.

**RoBERTa (约0.35):** 星形度最低. 这很自然, 因为它的结构是复杂的网状, 而非星形.

**高阈值 (t=0.1):** 当把较弱的连接也考虑进来时.

**LLaMA (0.97):** 星形度急剧增高, 接近完美. 这说明LLaMA的结构就是一个纯粹的星形, 即使加上一些弱连接, 这个主导模式依然不变. 并且**没有中间层峰值**, 表明这种星形结构贯穿始终.

**Qwen & RoBERTa:** 星形度也增高, 并在中间层达到峰值. 这意味着它们的注意力结构在中间层变得"最有序"或"最像星形", 然后在后期又变得不那么像星形. 这与它们更动态, 更复杂的结构演化模式一致.
C. 度中心性 (Degree Centralization)
**含义:** 衡量图的注意力(度)是否集中在少数几个节点上. 值越高, 注意力越集中, 存在一个或几个"超级节点".

**LLaMA (0.54 → 0.61):** 中心性非常高, 并且随深度增加.

印证了"集中式"的特点. 注意力高度集中在BOS这个"超级节点"上, 并且越到深层, 这种依赖越强.

**Qwen (约0.5):** 中心性较高, 但比LLaMA略低, 且在网络中保持相对平稳.

这符合"分布式"的特点. 存在几个中心, 但没有一个像LLaMA的BOS那样具有绝对的统治地位.

**RoBERTa (0.08 → 0.22):** 中心性极低.

这是网状结构的典型特征. 注意力(度)相对均匀地分布在许多节点上, 没有明显的主导中心.
D. Signature Correlation Patterns
**含义:** 它分析了两个关键谱指标——**Fiedler值(连通性)** 和 **中心性**——之间的关系如何随注意力阈值(尺度)变化而变化.

**LLaMA (-0.95 → +0.34):**

**低阈值(强连接):** 强负相关(-0.95). 意味着连通性越强, 中心性越低. 这似乎反直觉, 但可以这样理解: 在一个星形图中, 唯一的"连通性"就是指向中心的边. 增加连通性(例如, 让外围点之间也产生连接)会破坏其完美的星形结构, 从而降低中心性.

**高阈值(弱连接):** 转为正相关.

**Qwen (-0.46 → +0.61):**

与LLaMA模式类似, 但强度不同. 从负相关转为强正相关. 这表明在宏观尺度上(考虑弱连接时), 它的连通性和中心性是同步增强的, 这可能是多个局部中心协同作用的结果.

**RoBERTa (+0.32 → -0.33):**

模式完全相反. 在低阈值下, 连通性和中心性是正相关的. 在高阈值下, 变为负相关. 这反映了其与解码器模型根本不同的网络组织原则.

**相关性符号翻转 (Correlation sign flip):**

**发现:** 所有三种模型都表现出符号翻转, 只是翻转的方向和强度不同.

它表明所有这些模型的注意力网络都是尺度依赖 (scale-dependent) 的. 你在微观尺度(只看强连接)下观察到的组织原则, 可能与你在宏观尺度(把弱连接也算上)下观察到的原则完全不同. 作者将这种符号翻转视为不同参考系的可靠数学"签名".

#### 4.4.3 KL散度分析

![图片](assets/dce262772bf4.png)

核心是回答一个问题: **这些Sink点(参考点)对维持注意力分布的"形状"和"稳定性"有多重要? 这种重要性在不同模型和不同网络深度中是如何体现的?**
A. ASink 属性 (Attention Sink Properties)

**平均KL减少量 (Avg. KL Reduction):**

**含义:** 在整个模型中, 移除沉降点平均会导致注意力分布的KL散度(相对于均匀分布)变化多少. 负值越大, 意味着移除沉降点造成的"破坏"越大, 分布变得越混乱.

**LLaMA (-0.0974) & RoBERTa (-0.1017):** 两者都有非常大的负值. 这说明在这两种模型中, 参考点对于维持注意力分布的结构和"有序性"都极其重要.

**Qwen (-0.0088):** 负值非常小, 几乎接近于零. 这表明在Qwen中, 移除单个参考点对整体注意力分布的破坏不大. 这与"分布式"的理念一致: 系统不依赖于任何单一的锚点, 移除一个, 其他的还能继续工作, 系统鲁棒性强.

**平均/最大沉降集中度 (Avg./Max. Sink Concentration):**

**含义:** 注意力沉降点(通常是少数几个)总共吸引了多少百分比的注意力.

**LLaMA (平均82.93%, 最大96.40%):** 集中度极高. 在某些层(L25), 几乎所有的注意力(96.4%)都涌向了那个唯一的沉降点(BOS). 这是"集中式"的最直接证据.

**Qwen & RoBERTa:** 集中度相对较低. 这说明在这些模型中, 注意力更分散, 没有被一个或几个点绝对主导.
B. 层级分布 (Layer-wise distribution)
这是本表格最关键的部分, 它揭示了KL减少量在网络不同深度下的**动态模式**, 从而为三种框架的演化路径提供了独特的"信息论签名".

**LLaMA (强负 → 中等负 → 强负):**

在LLaMA中, 参考点在所有层都扮演着至关重要的稳定角色. 移除它在任何阶段都会造成显著破坏. 这种跨层一致的强负KL模式是集中式框架的标志. 它是一个从头到尾都稳定工作的"全局坐标系".

**Qwen (正 → 混合 → 混合):**

**正KL的含义:** 在早期层, 移除沉降点反而让注意力分布变得更"尖锐", 远离了均匀分布. 这意味着早期的"沉降点"可能不是在扮演唯一的锚点, 而是在"平衡"注意力. 移除它后, 注意力会更极端地涌向其他新形成的局部中心.

**三阶段模式:** 早期探索(正KL), 中期建立局部网络(负KL), 后期进行整合(混合KL). 这描绘了一个动态的, 自组织的建构过程.

这是分布式坐标系最独特的签名: 早期层为正KL.

**RoBERTa (混合 → 强负 → 可变):**

**早期层(混合/弱):** 参考系的作用还不明显, 模型可能在进行初步的语义信息收集.

**中间层(强负KL):** KL减少量达到**峰值**. 这说明**中间层是双向式框架的"主战场"**. 在这里, 参考点(CLS/SEP)的作用最大, 模型正在进行关键的几何重组和信息整合. 这与费雪信息在中间层达到峰值的观察完美吻合.

**深层(可变):** 几何结构已经建立, 注意力模式可能根据顶层任务的需求变得更加多样化.

这是双向式坐标系的独特"U形"或"中间重"签名.
C. 架构启示 (Architectural Implications)
本部分是对上述发现的总结和升华.

**Sink 形成 (Sink formation):**

**LLaMA (跨层一致):** 参考系是一个静态的, 全局的结构.

**Qwen (逐层变化):** 参考系是一个动态的, 演化的结构.

**RoBERTa (强烈的中间层焦点):** 参考系的形成有一个关键的"相变"阶段, 发生在网络的中间部分.

**早期层上下文 (Early layer context):**

**LLaMA (强Sink焦点):** 从第一层开始, 世界就围绕着BOS旋转.

**Qwen (弱Sink形成):** 早期在探索, 没有固定的中心.

**RoBERTa (极弱Sink形成):** 早期几乎没有参考系的迹象, 注意力可能主要服务于局部语义关联.

#### 4.4.4 Fisher信息分析

![图片](assets/83bdda8df099.png)

Fisher信息范数()衡量了第  层的参数对最终注意力分布的敏感度. 简而言之, **哪一层的Fisher信息范数高, 哪一层就是塑造注意力几何的"主战场"**.
A. Fisher信息范数分布 (Fisher Information Norm Distribution)

**LLaMA (集中式):**

**峰值在早期层, 集中度极高.** 约60-80%的信息都集中在最初的少数几层.

这是支持"集中式"理论的证据. 它表明LLaMA的"参考框架"或"坐标系"是在网络的最开始就被**一次性地, 决定性地**建立起来了. 就像建房子先打好地基, 后续的层只是在这个固定的地基上添砖加瓦(进行内容处理), 而不再去修改地基本身. 早期层的参数被训练来产生星形注意力, 一旦形成, 这个结构就基本锁定了.

**Qwen (分布式):**

**多个峰值, 分布式, 集中度较低.**

Qwen的学习能力没有集中在任何一个单一阶段. 存在多个"主战场", 分布在网络的不同深度. 这完美地印证了"分布式"框架的理念: **坐标系的建立是一个持续的, 动态的过程**. 早期层可能建立一些初步的局部参考点, 中间层对它们进行整合和调整, 深层可能还在进行微调. 学习任务被分散到了多个层级.

**XLM-RoBERTa (双向式):**

**为什么不是早期层?** 因为早期层在进行初步的, 可能更偏向语义的上下文收集. 此时, 模型还未开始进行大规模的几何重组, 所以早期层参数对最终的几何结构不敏感.

**为什么是中间层?** 因为中间层是进行**关键信息融合和几何重组**的阶段. 正如表3的KL分析所示, 参考框架的重要性在中间层达到顶峰. 因此, 中间层的参数自然成为塑造最终注意力分布的主力军, 其Fisher信息也最高. 这为"双阶段计算"理论提供了最直接的支持.

**峰值在中间层.**

这同样是一个极其深刻的发现, 与RoBERTa的其他所有"签名"都完美契合.
B. 与层深度的相关性 (Correlation with Layer Depth)
这部分用统计数据量化了上述的分布模式.

**LLaMA (斯皮尔曼 ):**

Fisher信息与层深度的关系是一个**近乎完美的单调递减**. 也就是说, 几乎可以肯定地说, **层数越深, 该层对注意力几何的塑造能力越弱**. 这个极高的负相关性是"学习能力集中在最开始"的最强统计证据. (皮尔逊相关性-0.39稍弱, 说明衰减不是完美的直线, 但单调性极强).

**Qwen (皮尔逊 ):**

同样表现出**强烈的负相关**. 这说明Qwen的整体趋势也是学习能力随深度衰减. 但与LLaMA不同的是, 它的衰减不是单调的(因为有多个峰值), 而是呈现一种更复杂的, 但总体向下的趋势.

**XLM-RoBERTa (相关性不显著, ):**

Fisher信息与层深度之间**没有简单的线性或单调关系**. 它告诉我们, RoBERTa的学习动态**不是简单的"越深越弱"**. 它的学习能力分布曲线是"中间高, 两头低"的, 这种非单调的模式无法被简单的相关性系数捕捉, 因此相关性不显著. 这个"不显著"本身就是其复杂学习模式的一个证据.
C. 学习动态启示 (Learning Dynamics Implication)
这部分是对上述所有发现的最终概括.

**LLaMA:** 坐标系在**起点**建立.

**Qwen:** 坐标系在**整个深度中持续构建和调整**.

**RoBERTa:** 坐标系在**中段**进行关键的构造和重组.

#### 4.4.4 集中式参考系

集中式参考系出现在具有标准旋转位置嵌入(RoPE)的Decoder-Only架构中, 包括LLaMA, Mistral和Gemma. 图谱分析揭示了代数连通性(Fiedler值)与度中心性之间的强负相关性, 这表明集中式注意力结构优先考虑信息集中而非分布式连接. 这创造了一个具有主导中心节点的星形拓扑. KL散度分析一致地显示, 当移除 ASink 点时, KL减少量为负值, 这证明了参考点在维持几何稳定性方面的关键作用. Fisher信息度量显示出极端的早期层集中现象, 大约60%的总费雪信息集中在最初几层, 这量化了这些架构如何通过一个主导的早期参考点来建立其坐标系.

#### 4.4.5 分布式参考系

分布式参考系出现在具有修改过的位置编码方案的架构中, 例如Qwen 2.5和Phi-2的NTK-aware scaled RoPE. 图谱分析识别出一个独特的符号翻转相关性模式, 在Fiedler值和中心性指标之间, 从低阈值的负相关转变为高阈值的正相关, 这是分布式参考系的一个可靠签名. KL散度测量揭示了一个特征性的三阶段模式: 早期层为正KL减少量, 中间层为更强的负减少量, 晚期层为中等负值. 这个签名表明参考点在网络不同深度扮演不同功能. Fisher信息显示出较低的峰值集中度, 并在网络不同深度有多个显著峰值, 反映了一种建立坐标系的根本不同方法.

#### 4.4.6  双向参考系

双向参考系出现在具有绝对位置嵌入的Encoder架构中, 例如BERT和XLM-RoBERTa. 拓扑分析揭示了非常高的初始复杂性, 高Betti-1值表明早期层存在大量环/圈, 与Decoder模型形成鲜明对比, 图谱分析显示连通性在早期但非初始层达到峰值, 并且从早期到中间层的连通性急剧下降, 表明了快速的几何重组. 最独特的签名出现在KL散度分析中,它揭示了一个特征性的U形剖面, 在第一层和最后一层都有正的KL减少量, 证实了参考结构的双锚点性质. Fisher信息在中间层达到峰值, 而不是在开始, 表明了一种根本不同的信息分布方法.

**集中式参考系** 建立了一个强大的初始坐标原点, 随着网络加深逐渐容纳更细微的变换, 同时保持单一参考点. 负的几何-语义对齐度证实了这些参考系优先考虑几何结构而非语义关系.

## 5. 对未来Attention block的展望

这一节我们先来探讨一下设计Attn block的一些取舍. 首先我们将阐述一些系统层面的影响和限制. 然后再进一步展开讨论一些权衡以及基于各种约束下的一些新的Attn架构的探索

### 5.1 ASink对Attention的影响和限制

**Softmax**: ASink的产生很大程度上来自于Softmax归一化, 因此增加一些偏置(GPT-OSS), 或者在Attn函数前加一个Gate函数(Qwen-next), 以及构造softpick函数等方式, 试图避免归一化的影响. 但是另一方面通过EOT理论(参考[《大模型时代的数学基础(9)- SDPA和最优传输, 强化学习及信息几何的联系》](https://mp.weixin.qq.com/s?__biz=MzUxNzQ5MTExNw==&mid=2247494688&idx=1&sn=3d589f6d4be56ee372d5db4f8631b0cc&scene=21#wechat_redirect)), 通过Softmax函数生成的注意力权重的过程, 完全等价于一个单边的熵最优输运(One-Sided Entropic Optimal Transport, EOT)问题的精确解. 另一方面, 从softmax导数中自然产生的项, 按比例缩放后就是Fisher信息矩阵(FIM).这意味着反向传播隐式地计算并利用了注意力分布统计流形的曲率. 因此:

为了避免ASink而使用其它非Softmax函数的做法值得探讨

**位置编码**: 在本文的第四章介绍了ASink作为几何锚点的观点, 不同的位置编码方式可以产生不同的参考系和不同的拓扑结构. 同时现有的位置编码和语义在hidden-state上耦合在一起, 又和 ASink 耦合在一起. 因此

是否能够在hidden-dim上开一小块空间作为独立的可学习的位置编码, 以及关联到 ASink几何锚点的索引?

**模型层数的影响**:token 表征矩阵的秩会随着层数的增加而呈双指数级速度衰减, 最终所有 token 的表征会趋于一致. 这就是所谓的"秩崩溃(rank collapse)". 为了对抗这种固有的崩溃趋势, 模型必须学会一些机制来"减速", 而 ASink 就是其中之一. 因此:

我们需要某种机制对某些token在某些层之间的注意力进行一个恒等变化, 一方面是利用ASink来构建一些no-op的处理机制, 另一方面需要考虑如何在残差流中更好的维持这种恒等变化, 例如DeepSeek mHC

**长上下文过度混合的问题**: 对于超长上下文任务, 由于 tokenization 的限制, Transformer的注意力logits范围是有限的. 当上下文变得足够长时, 注意力分布必然会变得平滑(无法保持尖锐), 导致注意力"弥散"到所有 token上, 即"纯混合".

另一方面 Decoder-only Transformer 中, 由于因果掩码的存在, 序列开头的信息对后续所有 token 都有影响, 而序列末尾的信息很容易被"淹没". 那么:

在长序列任务中构建一定的分治策略, 基于block的sparse attention是否就成为一个必须要做的问题了? 同时构造一个语义相关 block并行的Attn是否可行?

**低精度稳定性**: 另一个问题是如果接纳并利用 ASink, 那么随之而来的Massive Activation的问题以及attn概率分布对参数敏感性的问题在低精度训练过程中都需要进行很好的处理, 那么

是否可以考虑在训练过程中专门加入对Attn block参数的Fisher信息矩阵F-范数的惩罚项? 通过这种方式达到低精度训练?

**和MoE相关的问题**: 如果接受ASink, 那么Massive Activations和Attention Score的分布对于MoE Router的影响也需要考虑, 或者更直接的说, 对于作为Sink token在MoE中如何选择Expert, 是否会对其它 token 的语义产生影响, 以及是否因为 ASink 会导致超级Expert的出现(即摘除这个Expert后模型整体性能下降严重), 这是一个值得探讨的话题.

结合拓扑和范畴论来看, 这一点和我以前提到的[《谈谈 Hierarchical Sparse Attention (HSA)》](https://mp.weixin.qq.com/s?__biz=MzUxNzQ5MTExNw==&mid=2247496275&idx=1&sn=5f8a8d8efff22033d3f2aed8a5844e53&scene=21#wechat_redirect)基于Nerve构造的方法也有一些类似之处, 下面我们来补充一些概念:

范畴 (Category): 一个范畴由两样东西构成: **对象 (Objects)** 和 **态射 (Morphisms)**.

**对象**可以看作是系统中的"节点"或"实体".

**态射**是对象之间的"箭头"或"关系",  表示从对象  到对象  的一个态射. 范畴最重要的规则是**组合律 (Composition)**: 如果存在态射  和 , 那么必然存在一个组合态射 .

单纯复形 (Simplicial Complex): 这是一个由点, 线段, 三角形, 四面体等"单纯形"粘合而成的几何对象.

0-单纯形: 一个点.

1-单纯形: 一条连接两个点的线段.

2-单纯形: 一个填充了三个点和三条边的三角形.

k-单纯形: 一个由 k+1 个点构成的"填充"体.

Nerve 构造 (The "Translation" Rule): Nerve 的作用在于它如何将范畴的组合规则映射到几何体的构建上.某种意义上来说, 我们也可以对Attn做Nerve构造.

构建 0-单纯形 (点): 范畴中的**每一个对象**都成为单纯复形中的一个**点**.

构建 1-单纯形 (线段): 范畴中的**每一个态射** 都成为连接点  和点  的一条**线段**.

构建 2-单纯形 (三角形): 这是最关键的一步. 当范畴中存在一个**可组合的态射对** 时, 我们就在点 , ,  之间填充一个**三角形**. 这个三角形代表了"从 A 经过 B 到达 C" 这一整条路径的结构.

构建 k-单纯形: 同样地, 当范畴中存在一条长度为  的**可组合态射链** 时, 我们就填充一个 k-单纯形.

简单的来说标准注意力计算了所有  的直接连接 (1-单纯形), 这是一个稠密图. Nerve 告诉我们, **可组合的路径**是更高阶的结构. 例如前面的算法通过锚点  的 **2-单纯形路径**: . 这样就完成了可稀疏的Attention计算. 并且可以进一步推广到更高阶的场景.

### A.1 拓扑数据分析概述

首先我们来介绍一下拓扑数据分析(TDA)的一些基本概念.
持久化同调(Persistent Homology,PH)
持久化同调是TDA的基础工具之一. 油管上有一个很简单的介绍视频《Introduction to Persistent Homology 》[18], 另外《Persistent homology-a survey》[19]也是持久化同调的经典综述.

简而言之, PH的核心思想是"多尺度分析", 想象一下, 在每个数据点上放一个逐渐变大的球. 当球很小时, 点是孤立的; 随着球变大, 它们开始连接, 形成连通块, 甚至可能包围出"环"或"空洞". 如果继续变大, 这些"环"或"空洞"又会被填满而消失. PH就是记录这些拓扑特征(连通块、环、空洞)在哪个尺度下 **"诞生"**, 又在哪个尺度下 **"死亡"**. 一个"寿命"很长的特征, 通常被认为是数据中稳定、重要的结构; 而"寿命"很短的特征, 很可能只是噪音.

![图片](assets/86a3fdab82ba.png)

但是背后的数学原理相对复杂,  代数拓扑(持久化同调的理论基础)是一个高度的抽象化工具, 它将几何对象的属性(如"洞")转化为代数对象(如同调群), 从而可以用代数的方法进行严谨的计算和证明. 然而, 这个过程是层层封装的:

条形码 (Barcode) :

**形式**: 每个持久性对 `(p, q)` 被画成一条从 `p` 到 `q` 的水平线段.

**优点**: 非常直观地展示了每个特征的**生命周期 (lifespan)**. "长条"对应重要特征, "短条"对应噪音.

**比喻**: 就像超市扫描商品条码, 长短不一的线条记录了信息.

持久性图 (Persistence Diagram, PD) :

**形式**: 每个持久性对 `(p, q)` 被画成二维平面上的一个点, 坐标就是 `(p, q)`.

**优点**: 1) 所有点都位于对角线 `y=x` 上方. 2) 点到对角线的**垂直距离 `(q-p)/sqrt(2)`** (或 `L-inf` 距离 `q-p`) 直接度量了特征的"显著性". 离对角线越远的点越重要. 3) 持久性图是一个多重集, 为其定义度量(如瓶颈距离, Wasserstein距离)提供了方便, 从而可以**定量地比较两个不同形状的拓扑特征**. 这是条形码不方便做到的.

**比喻**: 像一张星空图, 远离"地平线"(对角线)的星星更亮、更重要.

持久性图将拓扑特征表示为二维平面上的一个多重集(multiset). 但这种表示方式给统计和机器学习带来了几个天生的难题:

a) 难以定义"平均值" : 在PD所处的空间(一个由多重集构成的复杂空间)中, "平均"并没有一个简单自然的定义. 你不能简单地把点的坐标取平均, 因为不同PD中的点没有天然的对应关系, 甚至点的数量都可能不同.

b) 与经典ML算法不兼容: 一个PD不是一个向量. 我们不能直接把它喂给一个期望输入是 `[x1, x2, ..., xn]` 这种形式的算法.

c) 距离计算复杂: 虽然我们可以定义PD之间的距离, 如瓶颈距离(Bottleneck Distance)和Wasserstein距离, 但它们的计算通常是组合优化问题, 计算量较大, 尤其是对于有大量点的PD.

因此出现了持久性景观(Persistence Landscape, PL)的方法:

**形式**: 将PD中的每个点 `(p,q)` 变换成一个"帐篷"形状的函数, 然后将所有这些"帐篷"函数取逐点最大值, 得到一系列函数, 即为景观.

**优点**: 1) 景观是一组定义在上的函数, 属于一个函数空间(Banach空间). 这意味着我们可以直接使用成熟的统计学和机器学习工具(如计算均值、方差, 做PCA, SVM等), 而无需为PD这种特殊的数据结构发明新的算法. 2) 它天然地解决了PD中点的数量不一致的问题.

**比喻**: 把PD的"星空"变成了一系列"山脉轮廓". 我们可以很容易地计算平均山脉、山脉之间的距离等.

### A.2 TDA在深度神经网络中的应用

这一节的内容主要来自于论文《Topology of deep neural networks》[22].

传统视角:将DNN视为一个复杂的函数逼近器, 学习从输入到标签的映射.

TDA视角:将DNN视为一系列的映射. 关注的不是最终的, 而是数据流形在每一步的形状变化: . 网络的任务是逐步"解开"和"填补"流形的复杂拓扑结构.

论文发现, 拓扑变化主要发生在网络的深层. 初始层更侧重于几何变换(拉伸, 旋转), 为后续的拓扑变换做准备. 因此, 网络需要足够的深度来提供足够多的"步骤", 以便逐步完成从复杂到简单的拓扑简化过程. 数据拓扑越复杂, 就需要越深的网络.

这篇文章依赖**贝蒂数(Betti numbers)** 来量化拓扑, 因为它们是捕捉空间  形状的最简单的拓扑不变量, 具有直观的解释, 并且可以在持续同调的框架内对从  中采样的点云数据集进行计算.

### A.3 Topological Autoencoders

持续性同调是一种能够量化多尺度结构特征的数学工具, 但是如何将其引入到深度学习的训练和推理过程存在挑战. 拓扑特征的计算 (例如, 一个环是在某个尺度"突然"出现或消失的) 本质上是**离散**的, 无法直接计算梯度. 因此, 我们需要将离散的拓扑信息平滑地整合到连续的神经网络训练框架中. 论文《Topological autoencoders》[23]开展了这方面的工作, 主要包含两方面:

加入拓扑约束的自编码器(称为 TopoAE) 能够在不显著牺牲重建精度的情况下, 生成拓扑结构更优, 更具可解释性的低维数据表示.

提出一种新的损失函数能够衡量原始数据空间和降维后潜在空间之间的拓扑差异. 这种拓扑损失必须是可微分的, 这样才能将其作为正则项加入到自编码器的总损失中,并通过反向传播来指导网络参数的更新.
VR复形计算
给定一个有限度量空间 S (例如一个点云), 我们首先计算其距离矩阵  的 Vietoris–Rips 复形的持续同调. 通常做法是使用欧几里得距离计算 , 但持续同调计算和TopoAE的方法都不局限于任何特定的距离; 另外即使是不满足度量性质的相似性度量也可以成功地与  一起使用. 随后, 令 , 使得  为 Vietoris–Rips 复形.

我们用  表示对 Vietoris-Rips 复形进行的持续同调计算. 其结果是一个持久性图的元组  和一个持久性配对元组.

持久性图 (Persistence Diagram, ): 一系列的点 , 代表一个拓扑特征在尺度  诞生, 在尺度  死亡.

持久性配对 (Persistence Pairing, ): 一系列的单纯形索引对 , 它告诉我们是哪个单纯形  "创造"了这个特征, 又是哪个单纯形  "杀死"了这个特征.

给定一个最大维度 , 我们得到一个持久性图集合  和一个持久性配对集合 . 第 d 维持久性配对  包含与 d 维拓扑特征的创建和销毁相关的单纯形的索引. 我们可以认为每个配对代表了边的索引, 即那些被持续同调计算认为是"拓扑相关"的边.

渣注
大多数TDA的应用只关注持久性图本身, 直接使用持久性图上的坐标  是困难的, 因为这些坐标是通过一个复杂的、不可微的矩阵消元算法得到的. 作者意识到, 持久性配对  提供了这座桥梁. 配对中的索引  直接指向了构成整个计算过程基础的单纯形. 而这些单纯形, 最终又是由数据样本点和它们之间的距离决定的, 这是实现可微性的关键一步.

**从配对中选择索引** 在持久性配对中选择索引, 并将它们映射回两个顶点之间的距离. 然后调整这个距离来协调输入空间和潜在空间的拓扑特征. 对于 0 维拓扑特征, 只需考虑配对  中作为"销毁者"单纯形的边的索引就足够了. 每个索引对应于数据集最小生成树中的一条边. 具体操作:

对于0维同调, 持久性配对  中的每个配对都是一个 **(0-单纯形, 1-单纯形)** 的配对, 即 **(顶点, 边)**.

这个配对的意思是: 某个顶点(代表一个独立的连通分量)在被某条边连接到另一个分量时"死亡"了.

作者说: "只需考虑配对  中作为'销毁者'单纯形的边的索引就足够了." 这意味着, 我们根本不需要关心是哪个顶点"死亡"了, 我们只需要收集所有扮演"杀手"角色的边的索引.

这些边的索引集合, 正是该点云的MST的边集.

**和MST的关系:** 想象一下, 我们按照距离从小到大的顺序将边加入图中. 每当加入一条连接了两个不同连通子图的边时, 就对应于一个0维拓扑特征的死亡.最终, 所有导致连通分量数量减少的边, 恰好构成了该点云（视为一个全连接图）的最小生成树.

因此, 对于0维拓扑, "从配对中选择索引"这个看似复杂的操作, 最终被简化为: **计算点云的最小生成树, 并返回构成这棵树的所有边的索引.**
